# 一致性validators

## 验证项

- [ ] 学习目标与评价是否一致（每条目标都有对应的评价任务）
- [ ] 评价与活动是否一致（评价方式能与教学活动匹配）
- [ ] 活动是否支撑目标（活动内容直接服务于目标达成）
- [ ] 是否存在无效活动（活动与目标无关，或可有可无）
- [ ] 是否存在无法评价目标（有目标但没有对应的评价设计）
- [ ] 是否符合逆向设计（目标 → 评价 → 活动的设计顺序）
- [ ] 教学设计是否在 `source_files` 中登记对应课时教材参考解答
- [ ] 评价与活动引用的每个教材 `question_id` 是否均存在于教材参考解答
- [ ] 教材题的成功标准、预期回答、反馈要点和答案是否与教材参考解答一致
- [ ] 是否完成教材-练习册双资源盘点，且评价、当堂检测、作业、后移题均来自已登记题源
- [ ] 练习册题若未进入 `practice` 或 `homework`，是否在 `deferred_exercises` 中说明后移去向或取舍理由

---

## 验证流程

1. 对每条学习目标，找到对应的评价任务
2. 对每个评价任务，找到对应的教学活动
3. 检查是否存在无对应评价的"空目标"
4. 检查是否存在不支撑目标的"无效活动"
5. 检查整体的目标-评价-活动链条是否完整
6. 确认设计顺序符合逆向设计逻辑
7. 校验教材参考解答存在、`content_type` 和 `lesson_id` 匹配，并已通过教材问题解答validators
8. 按 `question_id` 核对教学设计中的教材题覆盖和答案基线；任一缺失或不一致均判定不通过
9. 校验 `resource_audit` 中的练习册题去向：进入 `practice`、`homework` 或 `deferred_exercises`，不得静默遗漏

---

## 验证结果

- 通过：目标-评价-活动链条完整一致
- 有条件通过：存在链条断裂，附修复建议
- 不通过：严重不一致，需退回重新设计
