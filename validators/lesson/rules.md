# 教学设计validators

## 验证项

- [ ] 是否采用双层教学设计（第一层课堂实施导航 + 第二层折叠的完整结构化设计）
- [ ] 第一层是否包含核心问题、时间流程、实施要点、核心提问、课堂练习与作业
- [ ] 第一层是否为教师可执行语言，且不出现稳定 ID 或题源后台字段
- [ ] 第二层是否完整包含 `meta、knowledge_analysis、objectives、assessment、problem_chain、lesson_flow、practice、homework、boardwork、consistency_matrix、quality_check`
- [ ] 两层核心问题、活动数量、活动顺序、总时长、课堂练习与作业是否一致
- [ ] 是否符合课时结构（导入 → 新知 → 例题 → 活动 → 总结）
- [ ] 是否符合认知推进（从已有经验出发，逐步提升认知层次）
- [ ] 是否符合小台阶原则（环节之间认知跨度合理，有脚手架）
- [ ] 是否嵌入评价（关键认知节点有形成性评价设计）
- [ ] 是否体现数学活动（学生有观察、操作、归纳、探究等真实数学体验）
- [ ] 是否支持分层（不同认知水平的学生有差异化的任务）
- [ ] 是否包含快反馈（每个活动或关键步骤后有反馈节点）
- [ ] 是否与学习目标一致（所有环节的目标指向清晰）
- [ ] 数学表达式是否全部使用 LaTeX 格式（`$...$` 或 `$$...$$`），无裸文本公式、分式、指数、方程

---

## 验证流程

1. 检查双层结构及两层必备栏目是否完整
2. 检查第一层是否无后台字段泄漏，第二层是否保留完整结构化字段
3. 比对两层核心问题、活动数量、顺序、时间、练习和作业
4. 检查课时结构是否完整
5. 分析各环节之间的认知递进关系
6. 检查脚手架设置是否充分
7. 确认评价任务是否嵌入关键节点
8. 判断活动设计是否体现真实数学体验
9. 检查分层任务设计是否合理
10. 确认反馈节点设置是否充分
11. 对照学习目标逐条校验一致性

---

## 验证结果

- 通过：所有检查项均满足
- 有条件通过：存在可优化项，附修改建议
- 不通过：存在严重问题，需退回重新设计
