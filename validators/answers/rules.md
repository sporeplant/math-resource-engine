# 参考答案validators

&gt; 验证参考答案文件是否符合 参考答案生成skills 的全部outputs规则
&gt; **数学零容忍：任何数学错误直接拦截**

---

## 验证项

### A 组：结构合规（硬性条件，一票否决）

- [ ] 文件元数据是否完整且正确（content_type: courseware_reference、lesson_id、lesson_name、source_files、created_at）
- [ ] 随机池选取记录是否存在且表格格式正确（层次、选取学生、选取依据三列）
- [ ] 板块A【ASK课堂提问参考答案】是否存在
- [ ] 板块B【例题与练习解答】是否存在
- [ ] 评分量规是否存在且表格格式正确（准确性、完整性、表达力三维度）
- [ ] 所有题目是否标注了 source_id、source_type、question_id
- [ ] **所有题目来源是否为knowledge中的真实题源（禁止LLM自行生成题目）**
- [ ] `source_files` 是否登记对应课时教材参考解答，且该文件存在、课时匹配并通过教材问题解答validators

---

### B 组：板块A质量（核心检查）

- [ ] 是否按基础层、中间层、拓展层正确分组
- [ ] 每个问题是否包含7要素：问题编号、所属活动、学生姓名、问题原文、分级备用提示、参考答案、评分要点
- [ ] 每个问题是否按“问题→分级备用提示→参考答案→评分要点”排列
- [ ] 直接对应教材任务的问题是否以同一 `question_id` 的教材参考解答为事实基线并保留答案来源
- [ ] 问题编号格式是否正确（ASK-B-XX、ASK-M-XX、ASK-E-XX）
- [ ] 学生姓名是否与分层提问skills的选取结果一致
- [ ] 参考答案是否拆分为清晰要点（3-5个要点）
- [ ] 评分要点是否明确可观察、可量化，每项配有具体分值
- [ ] 评分要点总分是否控制在 2-5 分
- [ ] 是否区分了"准确性"、"完整性"、"表达力"维度
- [ ] 基础层答案是否侧重事实性、记忆性内容
- [ ] 中间层答案是否侧重推理性、应用性内容
- [ ] 拓展层答案是否侧重开放性、创新性内容

---

### C 组：板块B质量（核心检查）

- [ ] 是否覆盖了教学设计 lesson_flow 中引用的所有教材例题
- [ ] 是否覆盖了教学设计 lesson_flow 中引用的所有教材练习
- [ ] 是否覆盖了教学设计 lesson_flow 中引用的所有习题库题目
- [ ] 台阶递进式核心题是否包含完整解题步骤（不仅是最终答案）
- [ ] 每个题目是否正确标注了 source_id、source_type、question_id
- [ ] 每个教材 `question_id` 是否都存在于教材参考解答
- [ ] 教材题是否复用教材参考解答且未改变数学结论、依据和答案来源
- [ ] 练习册题是否仍按练习册题源生成
- [ ] 证明题是否包含完整推理链，每步有依据
- [ ] 计算题是否展示关键计算步骤和最终结果
- [ ] 概念题是否包含准确定义和辨析说明
- [ ] 应用题是否包含建模过程、求解步骤、结论解释
- [ ] 解答语言是否使用规范数学术语

---

### D 组：数学准确性（零容忍，任何一项不通过直接拦截）

- [ ] 所有数学概念、定义表述是否准确无误
- [ ] 所有公式、定理引用是否正确无误
- [ ] 所有推理步骤是否逻辑完整、无断裂、无错误
- [ ] 所有计算过程是否正确、无计算错误
- [ ] 所有几何推理是否符合规范
- [ ] 所有数学符号使用是否正确（使用 LaTeX 格式）

---

### E 组：协作一致性（参考性检查）

- [ ] 学生姓名是否与分层提问skillsoutputs的选取结果完全一致
- [ ] 问题内容是否与教学设计中的问题完全一致
- [ ] 题目解答是否与习题分析skills的8维度分析一致
- [ ] 评分量规是否与评价设计skills的评分标准一致

---

### F 组：页面长度控制（参考性检查）

- [ ] 板块A是否控制在 3-4 页 A4 纸内
- [ ] 板块B是否控制在 2-3 页 A4 纸内
- [ ] 总长度是否不超过 6 页 A4 纸

---

## 验证流程

1. **执行 A 组结构检查**：任一项不通过则直接拦截
2. **执行 D 组数学准确性检查**：任一项不通过则直接拦截（零容忍）
3. **执行 B 组板块A质量检查**：记录违规数量
4. **执行 C 组板块B质量检查**：记录违规数量
5. **B组 + C组 违规 ≤ 3 项** → 有条件通过（附修复建议）
6. **B组 + C组 违规 ≥ 4 项** → 不通过
7. **执行 E组、F组参考性检查**：仅记录建议，不影响通过判定
8. **outputs验证报告**：通过/有条件通过/不通过 + 违规明细 + 修复建议

---

## 验证结果

| 结果 | 条件 | 处理 |
|------|------|------|
| **通过** | A组和D组全部通过，B组+C组违规 ≤ 3 项 | 进入下一环节 |
| **有条件通过** | A组和D组全部通过，B组+C组违规 ≤ 3 项（记录违规位置和修复建议） | 附修复建议，酌情进入下一环节 |
| **不通过** | A组任意一项不通过 **或** D组任意一项不通过 **或** B组+C组违规 ≥ 4 项 | 回退到参考答案生成skills |

---

## 回退规则

- 不通过 → 回退到 **参考答案生成skills**
- 连续 2 次不通过 → 记录并升级提醒
- 连续 3 次不通过 → 触发人工介入

---

## 特殊规则

### 数学零容忍原则

以下任何一项发现，**立即拦截，无需继续其他检查**：
- 数学概念定义错误
- 公式、定理表述错误
- 推理逻辑断裂或错误
- 计算结果错误
- 几何证明步骤错误

### 题源真实性原则

以下任何一项发现，**立即拦截，无需继续其他检查**：
- 发现题目不是来自knowledge/textbooks/练习册题库
- 发现 source_id、source_type、question_id 缺失或伪造
- 发现LLM自行生成题目（未标注真实题源）
- 教材参考解答缺失、课时不匹配、验证失败或缺少被引用的教材 `question_id`
