# 课件validators

> 验证课件是否符合 课件设计skills 的全部outputs规则
> **支持多课型适配**

---

## 课型分类与验证策略

在验证前，请先确定课件课型，验证规则将据此调整：

| 课型 | 主要内容 | 特殊验证要求 |
|------|---------|-------------|
| **几何证明课** | 性质/定理推导、几何证明 | 必须包含 mermaid 推理链、完整证明步骤 |
| **计算课** | 计算题、运算步骤 | 不需要 mermaid 推理链，注重计算过程展示 |
| **统计/数据课** | 数据收集、整理、图表、统计过程 | 不需要 mermaid 推理链，注重数据展示与过程 |
| **概念新授课** | 概念引入、辨析 | 不需要 mermaid 推理链，注重概念呈现 |
| **复习课** | 知识回顾、综合练习 | 灵活配置，依内容而定 |

---

## 验证项

### A 组：结构合规（硬性条件，所有课型通用）

- [ ] 标题页是否存在（第1页为章节目录页，含 `### 📐` 标题）
- [ ] 学习目标页是否存在（3-4条目标，提炼自分层目标）
- [ ] 所有页面标题是否含正确的 emoji 类型（📐/📖/✏️/📝/💡/🤔），不得省略
- [ ] 课堂小结是否为三问三答表格（基础/提升/拓展各一个问题），且三层均配有分层提问（每层一位学生）
- [ ] 课后作业是否存在（分必做/选作/挑战三类，不含难度数字）
- [ ] 每页之间是否使用 Typora 可直接渲染的 Markdown 水平线 `---` 分隔
- [ ] 页面数量是否在合理范围（10-20 页，不含特大章节可适当扩展）
- [ ] 所有需配图的页面是否使用 `![图注](./images/文件名)`；**禁止出现猜测的假图片文件名**
- [ ] 图片引用路径是否有效（只能为 `./images/文件名`），指向文件是否真实存在 - 此项检查由 **[图片资源validators](../../validators/images/rules.md)** 独立完成
- [ ] **课件中禁止出现「教师：」「学生：」等教师动作提示或学生操作指令**（课件是给学生看的投屏内容）

### A1 组：几何证明课专属（仅几何证明课强制执行）

- [ ] 所有证明题是否拆分为 题目页 → 提问环节页 → 按需备用提示页 → 推理链分析页（mermaid）→ 规范证明页
- [ ] 规范证明页中推理区块用空行分隔，无数字编号

### B 组：内容合规（软性条件）

- [ ] 每个问题页在问题之前包含同页可见材料，学生无需依赖上一页或教师口头补充即可明确作答对象与依据
- [ ] “这四类、各数字、该图、前一个环节、哪两个读数”等指代均在同页明确列出所指内容
- [ ] 教学设计要求课堂完成的每一道题是否都有独立 `📝` 任务页，并明确题号/小问、作答动作、限时、评分和产出；仅有提问或答案不计

- [ ] 推理证明步骤中无 emoji（仅几何证明课适用）
- [ ] 所有 mermaid 图节点数 ≤ 6，含图注说明核心结论（若有mermaid图时检查）
- [ ] 无 ASCII 字符画替代专业图示（梯形等简单图形可用，但推理链必须用 mermaid，仅几何证明课适用）
- [ ] 每页正文 ≤ 10 行（含空行，不含图片引用和表格）
- [ ] 每页多张图片必须横向并排排列，单张宽度 < 页面宽度的 25%，高度 < 页面高度的 50%，保持原始纵横比
- [ ] 学生活动标注了时间限制（格式：`（限时 X 分钟）`）
- [ ] 检测/练习页标注了评分标准（格式：`评分：...`）
- [ ] 每个活动页标题下方是否有产出形式标注（`✏️ 请在练习本上...` / `🗣️ 口头回答` / `🗣️ 小组讨论后回答`）
- [ ] 提问页格式为两行分写：先出现**问题行**，再独立另起一行出现`**【层提问】**：请一位对应层级同学……`行，不得合并为一行
- [ ] 每个 `ASK_*` 是否按正文顺序“问题→按需备用提示→答案/归纳”排列；同页分批出现由课件软件设置，Markdown 不写控制信息
- [ ] 最终课件是否不含 YAML front matter、HTML、围栏代码块、制作说明、控制标签、占位符、`ASK_*`、后台题源字段等非教学文字
- [ ] 问题页是否未提前显示提示、答案或教师参考预期
- [ ] 学生课件是否未出现 `教师参考预期`
- [ ] 学生课件不出现具体学生姓名；点名信息与本课时课堂提问调度稿一致
- [ ] 课件读取的教学设计必须为 `review_status: 审核通过`
- [ ] 课件不重新生成学习目标、评价任务、活动设计，只转化人工审核通过的教学设计

### C 组：认知合规（参考性检查）

- [ ] 无整页未分段的纯文字墙
- [ ] 复杂公式独占一行，不嵌入正文堆叠
- [ ] 操作提示用 `>` 引用块标注，与正文区分
- [ ] 没有盲目复制教材完整段落的页面
- [ ] 没有「教师」「学生」开头的教师/学生动作提示语

---

## 验证流程

1. **确定课型**：先读取教学设计，确定本课课型
2. **执行 A 组通用检查**：任一项不通过则直接拦截
3. **执行 A1 组课型专属检查**（仅几何证明课）
4. **执行 B 组逐项检查**：记录违规数量和位置（依课型跳过不适用项）
5. **B 组违规 ≤ 2 项** → 有条件通过（附修复建议）
6. **B 组违规 ≥ 3 项** → 不通过
7. **C 组作为参考性检查**：仅记录建议，不影响通过判定
8. **outputs验证报告**：通过/有条件通过/不通过 + 违规明细 + 修复建议

---

## 验证结果

| 结果 | 条件 | 处理 |
|------|------|------|
| **通过** | A 组全部通过，B 组违规 ≤ 2 项 | 进入下一环节 |
| **有条件通过** | A 组全部通过，B 组违规 ≤ 2 项（记录违规位置和修复建议） | 附修复建议，酌情进入下一环节 |
| **不通过** | A 组任意一项不通过 或 A1 组（几何证明课）不通过 或 B 组违规 ≥ 3 项 | 回退到课件设计skills |

---

## 回退规则

- 不通过 → 回退到 **课件设计skills**
- 连续 2 次不通过 → 记录并升级提醒
- 连续 3 次不通过 → 触发人工介入

---

## 与规则的关系

- 提问格式：先出现**问题行**，再独立另起一行出现`请一位对应层级同学回答`行，不得合并
- A 组检查通用结构，A1 组检查课型特殊要求，B 组检查质量细节
- 新章节的课件在保证 A 组结构的前提下，B 组内容可依章节特点适当调整
