# 教材问题解答validators

验证教材问题参考解答的覆盖完整性、题源真实性、教材顺序、数学准确性和认知边界。

## A 组：结构与题源硬检查

- [ ] `content_type` 是否为 `textbook_solution`
- [ ] 是否包含教材任务清单、参考解答、覆盖统计和例题-习题台阶分析
- [ ] 每题是否包含稳定 `source_id`、`source_type`、`question_id`、教材位置和教材顺序
- [ ] 每题是否包含任务类型、认知层级和答案来源
- [ ] 答案来源是否只使用 `教材原文`、`AI参考推导` 或 `AI参考推导补充`
- [ ] 原题是否来自指定教材文件且未被改写
- [ ] 图片引用是否能在教材原文同级图片目录中找到对应文件
- [ ] 台阶分析是否包含方法标签、全链扫描表、断崖分析和升格建议
- [ ] 台阶分析是否包含"建议性质声明"（不自动改写教学设计，需教师确认）

A 组任一项不通过，结果为不通过。

## B 组：覆盖与顺序硬检查

- [ ] 是否识别教材中全部显式问题和任务指令
- [ ] 是否覆盖全部栏目、例题设问、练习、习题分组及小问
- [ ] 任务清单条目与解答条目是否一一对应
- [ ] `question_id` 是否唯一
- [ ] `教材顺序` 是否连续且与原文一致
- [ ] 是否未按认知层级重排题目

B 组任一项不通过，结果为不通过。

## C 组：数学准确性硬检查

- [ ] 所有定义、概念和定理是否准确
- [ ] 所有计算结果是否正确
- [ ] 所有推理链是否完整且成立
- [ ] 所有证明步骤是否有充分依据
- [ ] 所有图形结论是否与图片和已知条件一致
- [ ] 所有符号、单位和结论表达是否规范
- [ ] 所有数学表达式是否使用 Markdown 可渲染 LaTeX 格式；正文和答案中不得出现未用 `$...$` 或 `$$...$$` 包裹的裸文本公式、分式、指数、方程、单位化数学表达

C 组任一项不通过，立即拦截。

## D 组：题型适配检查

- [ ] 计算与应用题是否给出必要过程和结论
- [ ] 证明题是否给出已知、求证、证明和依据
- [ ] 作图题是否给出步骤、依据和检验
- [ ] 操作探究题是否给出操作、现象、结论和理由
- [ ] 开放题是否给出参考要点和答案边界，且未制造唯一答案
- [ ] 调查实践题是否只给方案和表达框架，未虚构实际数据

D 组违规两项及以上，结果为不通过；一项违规为有条件通过并要求修复。

## E 组：认知边界与异常检查

- [ ] 主解法是否只使用本课及此前教材已经出现的知识
- [ ] 是否未引入超出教材进度的新概念或成人化术语
- [ ] 参考解答是否使用了本章核心知识点而非其他章节方法（运行 `python tools/check_solution_knowledge.py --chapter {章号}` 检查）
- [ ] 应用题、证明题和综合题是否至少命中一个本章核心关键词
- [ ] 脚本报告的未命中题目中，真问题（方法偏离）是否已修正
- [ ] 图片缺失、题目残缺或条件不明时是否暂停作答
- [ ] 正常题目是否未写审核状态或生成状态
- [ ] 暂停题是否写明异常原因且没有猜测性结论
- [ ] 台阶分析的方法标签是否在本章核心关键词范围内，未使用后续章节概念
- [ ] 台阶分析是否执行了全链扫描（未跳过中间栏目直接比对例题与B/C组）
- [ ] 升格建议数量是否 ≤ 1，最终例题总数是否 ≤ 2

E 组任一认知越界或猜题行为，结果为不通过。知识点一致性检查中，应用题/证明题/综合题方法偏离本章视为认知越界；纯计算题仅写运算步骤而未命中关键词的为正常误报，不拦截。

## 验证结果

validatorsoutputs：

- `result`: 通过、 有条件通过或不通过
- `checked_file`: 被检查文件
- `failed_rules`: 未通过规则
- `fix_suggestions`: 修复建议
- `rollback_target`: 教材问题解答skills

回退目标固定为“教材问题解答skills”。
