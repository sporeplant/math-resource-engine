# 活动validators

## 验证项

- [ ] 是否以学生行为为中心（活动描述的是学生的操作，而非教师的讲解）
- [ ] 是否存在真实数学活动（学生有具体的数学操作或思维过程，而非仅听讲）
- [ ] 是否存在认知推进（活动过程中学生的认知水平有所提升）
- [ ] 是否符合小台阶原则（活动步骤之间递进合理，不存在过大的认知跨度）
- [ ] 是否包含快反馈（每个活动或关键步骤后有反馈节点）
- [ ] 是否存在伪探究（活动看似探究，实则以问答形式灌输结论）
- [ ] 是否存在教师单向灌输（教师讲授时间过长，学生缺乏参与机会）
- [ ] 活动顺序是否按照教材原文文本的先后顺序（练习和习题除外）
- [ ] 每个活动是否标注了 `教材对应位置`

---

## 验证流程

1. 逐项检查活动描述是否以学生为主体
2. 判断活动是否构成真正的数学体验
3. 分析活动步骤之间的认知递进关系
4. 检查反馈节点设置是否合理
5. 识别是否存在伪探究或灌输倾向
6. 验证活动顺序与教材原文文本顺序的一致性（详见下方"教材顺序验证流程"）

---

## 验证结果

- 通过：所有检查项均满足
- 有条件通过：存在个别可优化项，附修改建议
- 不通过：存在严重问题，需退回重新设计

---

## 教材顺序验证流程

### 前置条件

验证需要以下文件：
- 教学设计文件（含 `## lesson_flow` 节，活动均标注 `教材对应位置`）
- 教材原文文件（对应章节的教材原文，位于 `knowledge/textbooks/`）

### 验证步骤

1. **解析教材原文顺序**：提取教材原文中的 `##` 级标题，按出现顺序排列，排除"练习""习题"（含"A组""B组"等下级标记）标题，得到教材内容模块顺序列表 `T = [t₁, t₂, ..., tₙ]`

2. **解析活动顺序**：从教学设计 `## lesson_flow` 节中提取所有活动，按出现顺序排列，每个活动读取其 `教材对应位置` 字段值

3. **筛选教材关联活动**：从活动列表中筛选出 `教材对应位置` 值在教材内容模块列表 T 中的活动（即直接对应教材正文模块的活动），得到序列 `A = [a₁, a₂, ..., aₘ]`

4. **构建位置索引**：为每个教材内容模块分配序号（1, 2, ..., n）。将每个教材关联活动 aᵢ 映射为其对应教材模块的序号 pᵢ

5. **单调性检查**：验证序列 `P = [p₁, p₂, ..., pₘ]` 是否为非严格递增（即 `p₁ ≤ p₂ ≤ ... ≤ pₘ`）
   - 若 P 为非严格递增 → 通过
   - 若存在 `pᵢ > pᵢ₊₁` → 不通过，标记倒置位置和涉及的教材模块

6. **遗漏检查**（可选）：检查是否遗漏了教材原文中的关键模块（如整个"观察与思考"被跳过）

### 验证结果

- 通过：所有教材关联活动的顺序与教材原文顺序一致
- 不通过：存在倒置，列出倒置的活动编号和对应的教材模块名称，要求重新排序
