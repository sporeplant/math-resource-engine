# 复习讲义validators

## 验证对象

`/复习讲义` 工作流生成的复习讲义 Markdown 文件。

---

## A 组：硬性结构检查

- [ ] 文件位于 `outputs/g8-reviews/` 目录下。
- [ ] 文件包含 YAML front matter。
- [ ] `content_type` 为 `review_lesson`。
- [ ] `command` 为 `复习讲义`。
- [ ] `source_files` 至少登记 2 个来源讲义文件。
- [ ] 正文包含 `## 题目信息总览` 或 `## 题目信息总览表格`。
- [ ] 题目信息总览表格包含 4 列：`题目ID`、`知识考察点`、`难度`、`入选理由`。

任一项不满足，验证结果为不通过。

---

## B 组：题目与来源检查

- [ ] 正文题号使用 `1．`、`2．` 形式，且全讲义连续编号，不重置。
- [ ] 题目信息总览中的题目数量与正文题号数量一致或不少于正文题号数量。
- [ ] 题目 ID 能体现来源讲义，如 `03讲-...`、`04讲-...`。
- [ ] 合并两份讲义时，题目选择量应尽量均衡；数量差距超过 2 题时至少给出警告，超过 4 题判为不通过。
- [ ] 不得出现明显违背规则的表述，如“合并知识点”“整合为5个知识点”“压缩为5个知识点”。

---

## C 组：图片引用检查

- [ ] Markdown 图片路径必须使用同级 `images/` 目录。
- [ ] 禁止绝对路径、跨目录路径、空图片路径。
- [ ] 被引用图片应存在于讲义同级 `images/` 目录。

图片路径违规判为不通过；图片文件缺失可按实际情况判为不通过或有条件通过。

---

## 验证流程

1. 读取讲义 Markdown 文件。
2. 解析 YAML front matter，检查产物类型、命令和来源文件。
3. 检查outputs路径是否位于 `outputs/g8-reviews/`。
4. 提取正文题号，检查连续编号。
5. 提取题目信息总览表格，检查列名、题目 ID 和来源分布。
6. 检查两份来源讲义的题量均衡情况。
7. 检查图片引用路径和文件存在性。
8. outputs验证报告：通过 / 有条件通过 / 不通过。

---

## 验证结果

- 通过：A 组全部通过，B/C 组无错误，最多存在轻微提示。
- 有条件通过：A 组全部通过，B/C 组仅存在可修复警告。
- 不通过：A 组任一项失败，或存在题号断裂、严重来源失衡、图片路径违规、规则硬红线违背。

---

## 配套脚本

配套脚本：`tools/validate_review_handout.py`

推荐调用：

```powershell
python tools/validate_review_handout.py outputs/g8-reviews/03-04复习讲义.md
```
