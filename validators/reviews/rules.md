# 复习讲义validators规则

## 1. 基础结构

- 必须包含 YAML front matter。
- `content_type` 必须为 `review_lesson`。
- `command` 必须为 `复习讲义`。
- 必须包含：讲标题、知识点、例题讲解、当堂练习、课后作业。
- 题号必须从 `1．` 连续编号，不重置。

## 2. 题量

- 当堂练习必须包含全部 题型N 典例，不得遗漏任何典例。
- 课后作业数量由用户指定，若用户未指定则默认从变式池中选取 60% 左右（向上取整）。
- 每个知识点必须至少有 1 道例题。

## 3. 题型与难度

- 至少包含基础题、中等题、中等偏难题，不得全部集中在同一难度。
- 题型和难度分布应贴近源讲义原文比例。

## 4. 对比表格

- docs末尾必须包含「原始数量与选用数量对比」表格。
- 对比表格必须包含字段：`来源`、`类别`、`原始数量`、`选用数量`、`备注`。
- 对比表格行数必须与统计项一致（含合计行）。
- 最后一个数据行必须为合计行。
- 合计行中原始数量合计 = 知识点数 + 即学即练题数 + 全部典例数 + 全部变式数。
- 合计行中选用数量合计 = 知识点数 + 例题数 + 当堂练习题数 + 课后作业题数。
- 对比表格不得包含原有的「题目信息总览」表格（题目ID / 知识考察点 / 难度 / 入选理由）。

## 5. 来源与图片

- `source_files` 必须登记全部源讲义文件。
- 所有图片必须使用 `images/文件名`，且文件必须存在于outputs文件同级 `images/` 目录。
- 禁止引用 `knowledge/.../images`、绝对路径或空图片路径。
