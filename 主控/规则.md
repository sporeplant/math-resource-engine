# 初中数学教学资源系统全局规则（规则.md）

# 文件定位

本文件定义：

# 全系统最高优先级规则（Global Rules）

所有：

- Main Agent
- Workflow
- Skills
- Validators
- Sub Agents

必须严格遵守本文件规则。

任何模块不得覆盖、忽略或绕过本文件。

------

# 第一部分：系统核心原则

## 规则1：新课标优先原则

所有教学资源必须以：

# 《义务教育数学课程标准（2022版）》

为最高依据。

------

# 必须体现

- 核心素养
- 数学活动
- 数学思维
- 学业要求
- 教学评一致性

------

# 禁止

禁止：

- 脱离课标生成
- 经验主义教学设计
- 模板化套用
- 纯考试导向设计

------

# 第二部分：教学评一致性规则

## 规则2：教学评一致性（最高核心规则）

所有资源必须保持：

# 学习结果

↕
评价任务
↕
学习活动

三者一致。

------

# 学习结果要求

学习结果必须：

- 可观察
- 可评价
- 可检测
- 对应核心素养
- 体现数学活动

------

# 评价任务要求

评价任务必须：

- 能直接检测学习结果
- 提供明确评价证据
- 检测学生真实思维过程

------

# 学习活动要求

学习活动必须：

- 服务评价任务
- 支持学习结果达成
- 体现数学思维推进

------

# 禁止

禁止：

- 目标与活动脱节
- 活动与评价脱节
- 评价无法检测目标
- 先活动后评价

------

# 第三部分：逆向设计规则

## 规则3：逆向设计原则

系统必须遵循：

# 学习结果

→ 评价证据
→ 学习活动

逻辑。

------

# 禁止

禁止：

# “先设计活动，再补目标”

------

# 强制要求

所有学习活动：

必须能够回答：

# “该活动如何帮助学生完成评价任务？”

------

# 第四部分：数学学科真实性规则

## 规则4：数学真实性原则

所有资源必须体现：

- 数学抽象
- 数学推理
- 数学建模
- 数学表达
- 数学应用

------

# 禁止

禁止：

- 伪探究
- 形式化活动
- 空洞情境
- 无数学本质活动
- 假合作学习

------

## 规则5：数学表达规范

所有数学表达必须：

- 符号规范
- 推理完整
- 逻辑严谨
- 语言准确

------

# 禁止

禁止：

- 数学概念错误
- 推理跳步
- 非规范符号
- 口语化数学表达

------

# 第五部分：学习结果规则

## 规则6：学习结果可评价

所有学习结果必须：

- 可观察
- 可评价
- 可形成评价证据

------

# 禁止行为动词

禁止：

- 掌握
- 理解
- 了解
- 熟悉
- 提高
- 培养
- 激发兴趣
- 感受

------

# 推荐行为动词

优先使用：

- 判断
- 解释
- 描述
- 归纳
- 推理
- 建立
- 验证
- 表达
- 分析
- 建模

------

# 第六部分：核心素养规则

## 规则7：核心素养真实性

所有核心素养必须：

# 来源于真实数学活动。

------

# 禁止

禁止：

- 空泛素养标签
- 无活动支撑素养
- 模板化素养套用

------

# 素养内容对应规则

| 内容领域 | 优先核心素养 |
| -------- | ------------ |
| 几何证明 | 推理能力     |
| 函数     | 模型观念     |
| 统计概率 | 数据观念     |
| 图形变换 | 空间观念     |
| 应用问题 | 应用意识     |

------

# 第七部分：认知分层规则

## 规则8：默认认知分层

所有教学资源必须：

# 默认包含认知分层。

禁止：

# “统一认知要求”

------

# 分层结构要求

系统默认生成：

| 层级   | 认知要求       |
| ------ | -------------- |
| 基础层 | 能观察、能辨认 |
| 中间层 | 能解释、能归纳 |
| 拓展层 | 能分析、能创造 |

------

# 强制要求

必须：

- 允许低起点参与
- 保证中层学生递进
- 提供高阶挑战任务

------

# 第八部分：教学活动规则

## 规则9：学生主体原则

所有活动必须以：

# “学生学习行为”

为中心。

------

# 活动必须体现

- 观察
- 操作
- 比较
- 归纳
- 猜想
- 验证
- 推理
- 表达

中的至少一种。

------

# 禁止

禁止：

- 满堂灌
- 教师单向讲授
- 无目标合作学习
- 纯流程型活动

------

## 规则10：活动真实性

所有活动必须：

# “真实推动学生思维发展”

------

# 禁止

禁止：

- 假讨论
- 假探究
- 假发现
- 形式化展示

------

# 第九部分：反馈与评价规则

## 规则11：快反馈原则

系统必须：

提供：

- 高频反馈
- 即时反馈
- 过程性反馈

------

# 必须反馈

## 学业反馈

包括：

- 观察正确性
- 理解准确性
- 推理合理性
- 方法优化建议

------

## 状态反馈

包括：

- 参与变化
- 表达变化
- 思维投入变化
- 自主性变化

------

# 禁止

禁止：

- 仅关注结果对错
- 长周期无反馈
- 单一结果评价

------

# 第十部分：工作流规则

## 规则12：禁止跳过审核

所有资源：

# 未通过审核禁止输出。

------

# 必须审核内容

- 学习结果
- 评价任务
- 学习活动
- 习题解析
- 课件设计

------

## 规则13：审核失败回退

如果审核失败：

必须：

# 返回对应阶段重新生成。

禁止：

- 带错误进入下一阶段
- 强制继续流程

------

# 第十一部分：Skill行为规则

## 规则14：Skill职责隔离

每个Skill只能负责：

# 自身职责范围。

------

# 禁止

禁止：

- Skill越权修改
- Skill覆盖其他模块结果
- 未授权写入全局内容

------

## 规则15：统一输出协议

所有Skill输出必须：

- 结构统一
- 命名统一
- 可追踪
- 可审核
- 支持后续调用

------

# 第十二部分：资源工程化规则

## 规则16：资源可复用

所有输出必须：

- 可版本化
- 可追踪
- 可维护
- 可复用

------

# 禁止

禁止：

- 一次性自由文本
- 无结构内容
- 不可解析输出

------

# 第十三部分：全局教学法规则

系统必须长期遵循：

# “低起点、高目标、小台阶、快反馈”

教学原则。

------

# 系统要求

## 低起点

降低参与门槛。

------

## 高目标

提升高阶认知挑战。

------

## 小台阶

降低认知跨度。

------

## 快反馈

强化即时成功体验。

------

# 第十四部分：系统最终目标

本系统目标是：

# “构建符合学生认知发展的数学教学资源生产系统”

而不是：

# “一次性教案生成工具”。
