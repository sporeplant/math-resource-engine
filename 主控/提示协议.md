# Prompt协议

定义系统中各类Prompt的结构规范。

---

## 1. Main Prompt结构

```
[系统身份]
[用户输入]
[任务分类结果]
[知识注入]
[执行流程指令]
[输出格式要求]
[审核要求]
```

---

## 2. Skill Prompt结构

```
[Skill身份] — 你是谁，你的职责是什么
[输入数据] — 上游Skill的输出 + 注入的知识
[执行指令] — 具体要做什么
[生成原则] — Skill中定义的规则
[禁止事项] — 不可做的事
[输出模板] — 必须遵循的输出格式
[自检要求] — 完成后的自检项
```

---

## 3. Validator Prompt结构

```
[Validator身份] — 你是谁，你要审核什么
[审核对象] — 需要审核的内容
[检查项] — 必须逐项检查
[审核标准] — 什么算通过/不通过
[输出要求] — 审核报告的格式
```

---

## 4. 上下文拼接顺序

1. Main Layer 的全局上下文（任务类型、执行状态）
2. 当前阶段的知识注入（学情、课标、数学本质等）
3. 上游Skill的输出作为输入
4. 当前Skill的 技能.md 规则
5. 输出模板（输出合约.md 中对应部分）

---

## 5. Prompt长度限制

- Main Prompt 控制在 2000 token 以内
- Skill Prompt 控制在 3000 token 以内
- Validator Prompt 控制在 1500 token 以内
- 知识注入部分不超过总长度的40%
- 超出限制时，按 上下文注入.md 的优先级裁剪

---

## 6. Prompt禁止事项

- 禁止在Prompt中混入审核结果（审核结果应在审核阶段生成）
- 禁止在Prompt中写入与当前任务无关的知识
- 禁止在Prompt中预设答案或引导方向
- 禁止使用模糊指令（如"适当""合理"等无明确标准的词）
- 禁止在一个Prompt中包含多个Skill的职责
