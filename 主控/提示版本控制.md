# Prompt版本管理

## Prompt版本号

- 格式：`P-major.minor.patch`（如 `P-1.2.3`）
- major：Prompt结构或指令集的重大变更
- minor：新增指令或约束规则
- patch：措辞优化或示例更新

## 更新原因

- 高频错误触发：某一类错误在统计中持续高发，需修正Prompt指令
- 教师反馈驱动：教师明确指出的Prompt表述问题
- Validator新增：新增Validator需要对应Prompt增加校验提醒
- 知识库更新：知识库重大更新需Prompt同步调整引用方式
- 上下文优化：上下文注入方式或优先级需要调整
- 性能优化：Prompt长度过长影响响应速度，需精简

## 影响Skill

- 每个Prompt版本号对应一个或多个Skill
- 更新记录中需标明受影响的Skill列表
- 跨Skill的Prompt变更需通知所有相关Skill
- Skill独立更新Prompt时，仅修改对应Skill的版本号

## 回滚规则

- 新版本上线后发现P0/P1级别错误，立即回滚至上一版本
- 新版本导致审核通过率下降超过10%，触发自动回滚
- 回滚后记录回滚原因，并创建修复任务
- 回滚操作在5分钟内完成
- 同一Prompt在7天内最多回滚2次

## 版本记录

```
P-1.0.0 - 初始Prompt版本
P-1.1.0 - 增加"禁止使用不可观察动词"指令 (影响: 学习目标技能)
P-1.1.1 - 优化评价任务示例表述 (影响: 评价设计技能)
P-2.0.0 - 重构整体Prompt结构，增加Skill角色定义 (影响: 全部Skill)
```

## 版本存储

- 历史版本存放在 `versions/prompts/` 目录下
- 文件名格式：`{skill-name}_P-{version}.md`
- 每个版本记录创建时间、更新原因和变更内容
