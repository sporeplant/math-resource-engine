# 输出合约

所有最终输出必须是可读 Markdown，同时在文件头部提供机器可读 YAML front matter。

---

## 1. 输出目录

所有最终产物统一写入 `输出/`：

| 产物 | 文件名 |
|------|--------|
| 教学设计 | `输出/{课时名}_教学设计.md` |
| Markdown 课件 | `输出/{课时名}_课件.md` |
| 课堂提问参考答案 | `输出/{课时名}_课堂提问参考答案.md` |
| 输出图片 | `输出/images/{图片文件名}` |

---

## 2. YAML 元数据

每个输出 Markdown 必须以如下结构开头：

```yaml
---
content_type: lesson | courseware | question_reference
lesson_id: ""
lesson_name: ""
command: lesson | lesson-collab | courseware | courseware-collab
workflow_version: v2
review_status: draft | pending_human_review | human_approved | rejected
source_files: []
created_at: ""
---
```

### 字段规则

- `content_type`：输出类型。
- `lesson_id`：课时唯一标识，同一课时的教学设计、课件、课堂提问参考答案必须一致。
- `lesson_name`：课时名称，与文件名主体一致。
- `command`：生成该文件的命令。
- `workflow_version`：当前固定为 `v2`。
- `review_status`：人工审核状态。
- `source_files`：本产物直接读取的上游文件。
- `created_at`：生成时间，格式建议为 `YYYY-MM-DD HH:mm`。

---

## 3. 内容类型

| content_type | 生成命令 | 用途 |
|--------------|----------|------|
| lesson | `/lesson` 或 `/lesson-collab` | 教学设计 |
| courseware | `/courseware` | Markdown 课件 |
| question_reference | `/courseware` | 教师手持课堂提问参考答案 |

---

## 4. 稳定 ID 规则

关键对象必须携带稳定 ID，供下游引用和验证器检查。

| 对象 | ID 格式 | 示例 |
|------|---------|------|
| 基础层学习目标 | `LO-B-序号` | `LO-B-01` |
| 中间层学习目标 | `LO-M-序号` | `LO-M-01` |
| 拓展层学习目标 | `LO-E-序号` | `LO-E-01` |
| 评价任务 | `AS-层级-序号` | `AS-B-01` |
| 学习活动 | `ACT-层级-序号` | `ACT-M-01` |
| 课堂提问 | `ASK-层级-序号` | `ASK-E-01` |

层级代码：

- `B`：基础层
- `M`：中间层
- `E`：拓展层

---

## 5. 题源字段

任何题目、例题、练习、评价任务、作业题都必须携带：

```yaml
source_id: ""
source_type: textbook | exercise_bank | curriculum_repository
question_id: ""
```

禁止出现无来源题目。禁止将 LLM 生成题、仿题、类题、改写题作为新题使用。

---

## 6. 分层命名

全系统只允许使用以下层级名称：

- 基础层
- 中间层
- 拓展层

禁止使用旧的中间层别名作为系统层级名称。

---

## 7. 图片引用

任意 Markdown 文件引用图片时，只允许引用该文件同级目录下的 `images/` 子目录：

```markdown
<img src="./images/xxx.jpg" width="35%">
```

禁止：

- `![]()` Markdown 图片语法
- 跨目录引用知识库图片
- 引用旧资源目录图片
- 绝对路径

---
