# 跨Skill契约

定义各Skill之间的输入输出接口。

---

## 1. learning-outcome 输出给谁

**输出内容**：分层学习目标（基础层、中间层、拓展层）

**下游消费方**：
- 评价设计技能（作为输入）
- 活动设计技能（作为参考）
- 教学设计技能（作为输入）
- 一致性验证器（作为校验基准）

**输出格式**：按 技能.md 中的输出模板，每条目标以"能……"开头

---

## 2. assessment 输出给谁

**输出内容**：分层评价任务（目标-任务-成功标准）

**下游消费方**：
- 活动设计技能（评价方式约束活动设计）
- 教学设计技能（嵌入课时方案）
- 一致性验证器（校验目标-评价一致性）

**输出格式**：按 技能.md 中的输出模板，每项评价有明确成功标准

---

## 3. activity 输出给谁

**输出内容**：分层教学活动（活动名称、类型、步骤、反馈节点）

**下游消费方**：
- 教学设计技能（嵌入课时流程）
- 一致性验证器（校验评价-活动一致性）
- 课件设计技能（转化为PPT活动页）

**输出格式**：按 技能.md 中的输出模板

---

## 4. lesson-design 如何引用前置结果

- 直接引用 学习目标技能 的输出作为课时目标
- 直接引用 评价设计技能 的输出作为评价环节
- 直接引用 活动设计技能 的输出作为课堂活动
- 引用方式：在对应文件头部注明 source 和 lesson_id

---

## 5. validator 如何读取前置状态

- 从 会话记忆.md 读取当前课时状态
- 从 会话记忆.md 读取已完成的Skill列表
- 从对应Skill的输出文件读取待审核内容
- 从 元数据模式.md 读取目标文件的元数据
- 从 质量门.md 获取审核标准
