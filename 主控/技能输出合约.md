# 技能输出合约

本文件定义 Skill 之间传递内容的最小接口。最终落盘文件仍必须符合 `主控/输出合约.md`。

---

## 1. 通用返回结构

所有 Skill 输出必须包含：

- `content_type`
- `payload`
- `source_files`
- `must_be_valid_markdown: true`

`payload` 必须是 Markdown 片段，不输出 JSON 作为最终正文。

---

## 2. 分层与 ID

所有涉及分层的 Skill 必须使用：

- 基础层
- 中间层
- 拓展层

禁止使用旧的中间层别名。

所有关键对象必须带稳定 ID：

- 学习目标：`LO-B-01`、`LO-M-01`、`LO-E-01`
- 评价任务：`AS-B-01`、`AS-M-01`、`AS-E-01`
- 学习活动：`ACT-B-01`、`ACT-M-01`、`ACT-E-01`
- 课堂提问：`ASK-B-01`、`ASK-M-01`、`ASK-E-01`

---

## 3. 题源强约束

以下 Skill 涉及题目使用，必须执行题源探索并携带来源字段：

- 习题分析技能
- 评价设计技能
- 活动设计技能
- 教学设计技能
- 课件设计技能

每道题必须携带：

```yaml
source_id: ""
source_type: textbook | exercise_bank | curriculum_repository
question_id: ""
```

禁止：

- LLM 自行生成新题
- 改写后当作新题使用
- 仿题、类题、变式题替代题库题
- 无来源题目进入输出

---

## 4. `/lesson` 上游片段

### 学习目标技能

输出：

- `content_type: objectives`
- 三层目标
- 每条目标带 `LO-*` ID

### 评价设计技能

输出：

- `content_type: assessment`
- 每项评价带 `AS-*` ID
- 每项评价引用 `aligned_objective`
- 涉及题目时带 `source_id/source_type/question_id`

### 活动设计技能

输出：

- `content_type: activities`
- 每个活动带 `ACT-*` ID
- 每个活动引用目标和评价任务
- 每个活动包含反馈节点

### 教学设计技能

输出：

- `content_type: lesson`
- 完整课时设计 Markdown
- 总时长不超过 35 分钟
- 只标注提问层级，不写学生姓名

---

## 5. `/courseware` 下游片段

### 分层提问技能

输出：

- `content_type: question_reference`
- 每个提问带 `ASK-*` ID
- 学生姓名来自 `学生近期数据/学生成绩数据.md`
- 同一课时同一学生不重复，除非该层人数不足并记录原因

### 课件设计技能

输出：

- `content_type: courseware`
- Markdown 课件
- 使用 `./images/{文件名}` 引用图片
- 不重新生成目标、评价、活动，只转化人工审核通过的教学设计

---

## 6. 验证器输出

验证器输出必须包含：

- result: `通过 | 有条件通过 | 不通过`
- checked_file
- failed_rules
- fix_suggestions
- rollback_target

验证器不得修改 `review_status` 为 `human_approved`。
