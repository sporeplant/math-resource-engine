# 执行策略

定义系统执行过程中的各项限制条件。

---

## 1. Skill最大上下文长度

| Skill | 最大上下文长度 |
|-------|---------------|
| 学习目标技能 | 3000 tokens |
| 评价设计技能 | 3000 tokens |
| 活动设计技能 | 3500 tokens |
| 教学设计技能 | 4000 tokens |
| 教材分析技能 | 3000 tokens |
| 习题分析技能 | 3000 tokens |
| 课件设计技能 | 2000 tokens |

---

## 2. 并行执行限制

- 同时最多执行3个独立Skill
- 独立Skill：教材分析技能, 习题分析技能
- 串行链中的Skill不并行
- Validator不并行（同一时间只执行一个审核）
- 多课时批处理时，最多同时处理2个课时

---

## 3. Validator优先级

1. 一致性验证器（最终校验，优先级最高）
2. 教学设计验证器（设计方案审核）
3. 活动验证器（活动质量审核）
4. 评价验证器（评价质量审核）
5. 学习目标验证器（目标质量审核）

优先级高的Validator在资源竞争时优先执行。

---

## 4. 强制中断条件

- 检测到数学严重错误 → 立即中断当前Skill，回退
- 同一节点回退超过3次 → 强制中断，触发人工介入
- 上下文长度超出最大限制且压缩后仍超限 → 中断，拆分任务
- 用户主动发送中断指令 → 立即中断，保存当前状态

---

## 5. 人工接管条件

- 同一节点回退超过3次
- 数学严重错误二次出现
- 知识库冲突无法自动解决
- 用户主动要求人工审核
- 系统检测到未知异常
