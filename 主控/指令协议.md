# 命令协议

定义用户指令从输入到发布的完整命令流。

---

## 命令流

```
用户指令
   │
   ▼
┌─────────────────────────────────┐
│ 1. 任务识别                      │
│    task_classifier               │
│    判断任务类型（目标/评价/活动等）  │
└──────────────┬──────────────────┘
               │
               ▼
┌─────────────────────────────────┐
│ 2. 上下文注入                    │
│    context_injection            │
│    按任务类型注入对应知识           │
│    按 context_priority 裁剪内容    │
└──────────────┬──────────────────┘
               │
               ▼
┌─────────────────────────────────┐
│ 3. Skill调用                     │
│    按 pipeline 顺序调用 Skill      │
│    每个 Skill 完成后自检           │
│    输出写入对应文件                │
└──────────────┬──────────────────┘
               │
               ▼
┌─────────────────────────────────┐
│ 4. Validator审核                 │
│    调用对应 Validator             │
│    审核通过 → 进入下一步           │
│    审核不通过 → 回退到 Step 3     │
└──────────────┬──────────────────┘
               │
               ▼
┌─────────────────────────────────┐
│ 5. 输出发布                      │
│    一致性验证器 最终校验 │
│    更新 lesson_lifecycle 状态      │
│    按 output_pipeline 发布        │
│    归档到 release 目录             │
└─────────────────────────────────┘
```

---

## 命令流规则

- 每个步骤完成后更新 会话记忆.md
- 步骤之间通过标准接口传递数据
- 异常时按 错误处理.md 处理
- 人工介入时暂停自动流转
