# 复习课讲义整理检查清单

## 生成前

- [ ] 用户已明确指定输入讲义，不做默认假设。
- [ ] 用户已明确指定outputs文件名。
- [ ] 已读取 `skills/reviews/SKILL.md`。
- [ ] 已读取 `validators/reviews/rules.md`。
- [ ] 已读取所有来源讲义原文和必要图片目录。

---

## 内容结构

- [ ] 知识点数量等于源讲义已有知识点数量，不合并。
- [ ] 专题型讲义按专题块、题型块或代表例题块保留结构，不强行改写成知识点。
- [ ] 每个已有知识点完整保留概念讲解文字。
- [ ] 每个知识点或专题块配 1 道代表例题。
- [ ] 当堂练习约 10 题。
- [ ] 课后作业约 10 题。
- [ ] 全讲义题号连续编号，不在模块之间重置。
- [ ] 题目信息总览覆盖正文题目。

---

## 题源与均衡

- [ ] 例题、当堂练习、课后作业均尽量覆盖两份来源讲义。
- [ ] 两份来源讲义的题目选择量尽量均衡；差距超过 2 题时需说明理由并优先调整。
- [ ] 若某一来源题量明显不足，可向另一来源倾斜，但不得长期偏向单一来源。
- [ ] 题目信息总览中的题目 ID 能追溯到来源讲义。
- [ ] 禁止虚构题目、改写题目或使用无题源题。

---

## 图片与格式

- [ ] 只复制实际被引用的图片到outputs目录同级 `images/`。
- [ ] 图片引用统一使用 `![](images/{文件名})`。
- [ ] 禁止绝对路径、跨目录路径和空图片路径。
- [ ] 题号使用 `1．`、`2．` 形式。
- [ ] 题目信息总览表格包含 `题目ID / 知识考察点 / 难度 / 入选理由` 四列。

---

## 生成后验证

- [ ] 运行 `python tools/validate_review_handout.py {outputs讲义.md}`。
- [ ] 验证结果为“通过”或“有条件通过”。
- [ ] 若为“有条件通过”，已处理或说明所有警告。
- [ ] 若为“不通过”，必须修复后重新验证。
