# 课件设计skills

## 1. 核心原则：严格遵从教学设计流程

> **课件是服务于教学设计落地的师生交互抓手，课件页面顺序必须严格对应教学设计的 lesson_flow 环节和活动顺序！**

- **课件结构必须 1:1 映射教学设计的 lesson_flow**（环节一 → 活动1 → 活动2 → ... → 环节五）
- **禁止打乱教学设计的环节和活动顺序**
- **每个教学设计的活动环节，在课件中必须有对应的页面序列**
- **教材对应位置必须在课件中明确标注**
- **教材原文材料必须进入课件**：凡课堂提问、作图、计算、判断所依赖的教材原文背景、数据表、统计表、方案表、图形、题干摘要，必须在问题出现前或同页转化为学生可见材料。禁止只引用教学设计摘要而省略教材材料锚点。

### 1a. 教材原文材料转化规则

课件生成时必须先读取对应 `knowledge/textbooks/{课时}.md`，并按教材原文顺序提取以下材料：

| 材料类型 | 课件处理要求 |
|---|---|
| 实际问题背景 | 用 2-3 句呈现问题来源、统计目的和待判断对象 |
| 原始数据 | 取值较多的数据必须以表格或图片形式呈现，不能只说“看教材数据” |
| 教材步骤 | 按教材原文步骤顺序拆页，保留关键计算式、区间、表格列名 |
| 统计表/方案表 | 频数表、频率表、阶梯方案表等必须进入课件，支撑后续提问 |
| 教材图片 | 读取真实图片文件名，复制或登记到 `outputs/images/` 后引用 |
| 练习题 | 练习/检测页可只出示编号和作答要求，但若题目依赖图表或背景，必须呈现对应图表或简要题干 |

每个结构化提问必须能回指到课件中的可见材料；若学生离开教材也无法作答，课件页必须补充教材材料。

### 1b. 单页独立作答门禁

- 每个问题所在页必须在问题之前呈现足以作答的题干、数据、图表、对象或过程。
- 不得把必要材料只放在上一页，也不得依赖教师口头补充。
- 出现“这四类、这些、各数字、该图、前一个环节、哪两个读数”等指代时，同页必须明确列出所指内容。
- 生成后逐页执行“隐藏上一页后，学生能否说清在问什么、依据什么作答”的检查；不能则回退补材料。

## 2. 课件格式

- 课件以 **Markdown docs** 形式outputs，适用于typora浏览后截屏到希沃白板课件中使用。
- 课件只能由 `/courseware-collab` 生成，且必须读取 `review_status: 审核通过` 的教学设计
- 课件不得重新生成学习目标、评价任务、活动设计，只能转化人工审核通过的教学设计
- 每页使用 Typora 可直接渲染的 Markdown 水平线 `---` 分隔
- 整体结构：**标题页 → 学习目标页 → [教学设计各环节对应的页面序列] → 总结页 → 作业页**
- 复杂例题必须拆分为题目、提问、推理分析和规范解答等清晰页面；推理关系用表格、列表或图片呈现。

---

## 2. 页面结构

- 每页聚焦 **1 个核心信息**

- 页面结构：**标题（含 emoji）+ 内容主体**

- 标题用 `###` 级别，突出本页关键词

- 内容控制在 **5-7 行以内**（不含图片和表格）

  - 不得包含教学对话（如 `教师引导：`、`学生回答：`）
  - 提问格式：先出现**问题行**，再独立另起一行出现`请[姓名]同学回答`行，不得合并

### 2a. 学习目标页规则

- **🎯 学习目标页显示 3~4 条目标**，不得超过 4 条
- 目标来源：从教学设计的 6 条分层目标（基础层2 + 中间层2 + 拓展层2）中提炼合并，而非逐条照搬
- 提炼原则：
  - 覆盖本课核心任务（知识框架、方法掌握、能力提升三个维度至少各 1 条）
  - 合并同维度目标（如同属知识建构的两条目标合并为一条）
  - 省略高阶延伸目标（拓展层的高阶目标不出现在课件目标页中，后续在对应活动页面自然呈现）
  - 语言转化为学生视角的"能够……"表达
- 格式示例：

  ```markdown
  ### 🎯 本课目标

  1. 能够画出四边形家族树，说出各图形的从属关系与对角线性质差异
  2. 能够区分性质定理与判定定理，掌握矩形、菱形的判定路径
  3. 能够运用特殊四边形的性质与判定完成推理证明
  ```

- 学习目标页之后不得再出现其他学习目标（每课件仅一张目标页）

## 3. Emoji 使用规则

每页标题 **必须** 包含 emoji，不允许省略。仅使用以下映射：

| 页面类型 | emoji | 说明 |
|---------|-------|------|
| 标题页 | 📐 | 章节标题 |
| 学习目标 | 🎯 | 本课目标 |
| 新知讲授 | 📖 | 定义、性质、定理 |
| 例题示范 | ✏️ | 教师例题演示 |
| 课堂练习 | 📝 | 学生独立练习 |
| 小结总结 | 💡 | 本课核心要点 |
| 思考讨论 | 🤔 | 学生思考/讨论问题 |

- 每页最多使用 **1 个 emoji**，放在标题中
- 内容正文中不添加装饰性 emoji
- 数学公式、推理步骤中绝不使用 emoji
- 没有完全匹配的 emoji 时，选择最接近的 emoji 类型，**不允许省略 emoji**

## 4. 数学可视化规则

- 抽象关系优先使用 Markdown 表格、分级列表或可直接显示的图片呈现。
- 最终课件禁止 Mermaid 和其他围栏代码块，避免 Typora 设置差异导致源码直接显示。
- 避免使用 ASCII 字符画替代专业图示
- 需显示实际几何图形的题目，必须引用 knowledge/ 下对应图片（详见 §15 图片引用规则）

---

## 5. 认知负荷控制

- 每页信息量控制在学生能在 2-3 分钟内消化完毕
- 重点内容用 **加粗** 强调，避免全文加粗
- 复杂内容分页呈现，每页一个小台阶
- 页面之间留出足够的视觉间距（段落间空行）
- 数学公式独占一行，不嵌入正文中堆叠

---

## 6. 课堂互动页规则

- 问题初始呈现：只显示**问题 + 分层提问对象**，不得提前显示提示、答案或教师参考预期。`ASK_*` 编号仅保留在教师参考答案中，不进入学生投屏正文。
- 同页遮盖呈现：排版合理时，问题、备用提示和答案/归纳可在同一页依正文顺序排列，由教师在课件软件中设置分批出现；Markdown 文件不记录动画或遮盖控制信息。
- 最终课件只保留学生应看到的教学内容。禁止 YAML front matter、HTML、围栏代码块、制作说明、控制标签、占位符、`ASK_*`、`source_id/source_type/question_id` 等非教学文字。
- 图片使用 Typora 原生 Markdown：`![图注](./images/文件名)`。
- 分页呈现：内容较长时，仍按问题页 → 备用提示页 → 答案/归纳页拆分，并重复同一 `ASK_*` 编号。
- `教师参考预期`只进入课堂提问参考答案，不进入学生课件。
- 提问采用**两行分写**格式：先出现**问题行**，再独立另起一行出现`请[姓名]同学回答`行，不得合并为一行
- 小组活动页：**任务说明 + 时间控制 + 产出要求**
- 练习/检测页：**题目 + 时间限制 + 评分标准**
- 互动页必须标注活动类型标签
- **选择题/判断题的提问禁止透露答案**——不得在提问中包含"为什么A/C/D是错的""为什么选B不对"等暴露正确答案的措辞。正确做法：`请给出你的答案并说明理由`

### 6a. 课堂练习/检测页规则

课堂练习/检测页专门用于学生限时独立完成教材题目，与例题示范页（✏️）区分。

**练习/检测页基本格式**：

当堂检查第一页**仅出示题目的编号位置和作答要求，不必初始显示题干内容**，方便学生翻开教材对应位置独立完成，节省课件空间，实现适当密集排版。

```markdown
### 📝 练习/检测

✏️ 请在练习本上完成

教材练习第 X 题（来源：xxx）

（限时 X 分钟）
评分：步骤①得X分，步骤②得X分
产出：xxx
```

**参考答案页**：所有课堂练习/检测题目必须在课件中提供参考答案页，放在对应练习题目页之后，方便学生订正。

**任务显性规则**：教学设计 `practice` 或 `lesson_flow` 中标为“完成”的每一道课堂题，课件必须设置独立 `📝` 任务页，明确题号/小问、作答动作、限时、评分和产出。仅在提问页、答案页或教师参考答案中出现题号，不算已布置课堂练习。

```markdown
### 📝 参考答案

**第X题**：[简要解题过程与最终答案]
```

**参考答案编写要求**：
- 证明题：含完整推理链，每步有依据
- 计算题：展示关键计算步骤与最终结果
- 概念题：给出定义要点与辨析要点
- **参考答案页紧随题目要求页之后**，方便学生订正
- **参考答案适当密集排版**：同一张参考答案页可放多道题的答案，通过题号分隔，节省翻页次数
- 参考答案的数学符号使用 LaTeX 格式，步骤规范、逻辑清晰

**评分标准格式要求**：
- 客观题（选择、填空）：`评分：每题X分，共X题，满分X分`
- 主观题（证明、计算）：`评分：步骤①得X分，步骤②得X分，满分X分`

## 7. 反馈页规则

- 设置专门的"检查理解"页面
- 反馈页包含 1-2 道评价任务
- 选择题形式的快速检测优先
- 反馈页仅展示题目和答案，不附带教学建议

---

## 8. 禁止事项

- **禁止打乱教学设计的 lesson_flow 的环节和活动顺序**
- **禁止课件页面序列与教学设计 lesson_flow 不一致**
- **禁止省略教学设计中明确有的活动在课件中没有对应页面**
- 禁止一页放入过多知识点（超过 7 行正文）
- 禁止直接复制教材完整段落作为页面内容
- 禁止与教学设计中的核心板块内容完全重复
- 禁止使用无意义的装饰符号或分割线
- 禁止在推理证明步骤中使用 emoji
- 禁止连续 3 页以上纯文字无图示（不含图片引用）
- 禁止标题不含 emoji
- **禁止在课件中出现「教师：」等教师动作提示**
- **禁止在证明和计算步骤中使用数字标号**（如 `1. 2. 3.`），用空行分隔区块代替

## 9. Typora 可视化规则

- 关系和推理链使用 Markdown 表格、分级列表或真实图片呈现。
- 禁止 Mermaid、HTML、SVG 源码、ASCII 字符画和其他可能直接显示源码的写法。
- 图片配一句图注说明核心结论。

---

## 10. 推理链总结规则

- 四步及以上步骤的例题/证明题，末尾必须附"推理链"（Reasoning Chain）
- 推理链格式：用 `→` 连接关键步骤，形成一条完整的推导路径
- 推理链放在标准证明之后，独占一个页面区间
- 推理链使用的语言：关键词 + 箭头，避免完整句子
- 例：`等边△ → BC = BE → AB = BE → 等腰△ABE → 底角75° → ∠EAD = 15°`

### 10a. 证明与计算格式规则

- 用**空行分隔**不同推理区块，每个区块为一个独立推理步骤
- 每个区块内部可包含多行 `∵`/`∴` 链式推理，区块间空行分隔
- 计算题中每切换一个三角形/图形，即为一个新区块，用空行分隔
- 示例结构：

```markdown
$\because AB \parallel DC$

$\therefore$ 四边形 $AECD$ 是平行四边形

$\therefore AD = EC$

$\because AD = BC$

$\therefore EC = BC$

$\therefore \angle CEB = \angle B$
```

## 11. 课堂小结规则

- 课堂小结页必须包含**三问三答表格**（基础/提升/拓展各一个问题，不含答案）
- 三个问题必须对应三个层次的**分层提问**（基础层、中间层、拓展层各一个），逐行分写，两行格式
- 不得只提问一个层级

## 12. 单页聚焦规则

- 每页只传达 **1 个核心信息**
- 复杂内容必须拆分为多页，每页聚焦一个子步骤
- 学生活动/练习页：问题描述 + 操作指令 + 时间限制
- 答案/证明页：只呈现答案本身，不混入其他内容
- 使用 Markdown 水平线 `---` 作为页面分隔符

---

## 13. 学生活动指令规则

- 学生可使用的教学材料仅限于：**教材**、**练习册**、**练习本/纸笔**
- 禁止在课件中出现"学习单""检测卡""不同颜色笔""投影展示"等需要额外物料的表述
- 课件中的所有学生产出统一对应到练习本书写或口头回答，不得引入其他形式

- 每个学生活动必须有明确的 **时间限制**
  - 格式：`（限时 X 分钟）`
- 练习/检测题必须有 **评分标准**
  - 格式：`评分：步骤①得X分，步骤②得X分`
- 活动页底部标注 **产出要求**
  - 格式：`产出：写出完整证明过程`
- **每个活动页标题下方第一行必须标注产出形式**，学生据此知道该写还是该说：

| 产出形式 | 标注文字 | 适用场景 |
|:---|:---|:---|
| 练习本上书写 | `✏️ 请在练习本上完成` 或 `✏️ 尝试在练习本上…` | 画图、证明、计算 |
| 口头回答 | `🗣️ 口头回答` | 选择题、判断题、快速应答 |
| 小组讨论 | `🗣️ 小组讨论后回答` | 观察发现、开放性问题 |
| 教师演示 | 不标注（跟随板书即可） | 例题讲解、板书演示 |

- 产出形式标注独占一行，放在标题与题目描述之间

## 13. 习题选用规则

### 13a. 课堂练习/检测题目选用规则

课堂练习/检测题目优先选自教材原文，规则如下：

| 来源 | 选用规则 | 说明 |
|------|---------|------|
| 教材"练习"栏目 | **全部选用** | 教材原文中练习栏目下的所有题目，不得遗漏 |
| 教材"习题A组" | **选取 1 道** | 从习题A组中选取 1 道作为课堂练习/检测 |
| 教材"习题B组" | **选取 1 道** | 从习题B组中选取 1 道作为课堂练习/检测 |
| 练习册题库 | **补充选用** | 当教材来源题目总数不足 4 道时，从练习册题库对应章节补充 |

- 课堂练习/检测题目总数量 ≥ 4 道
- 后台题源字段只写入课堂提问参考答案；学生课件仅显示自然语言来源，如“教材A组第3题”。
- 课件题目来源标注格式：`（来源：教材练习第X题 / 教材习题A组第X题 / 教材习题B组第X题）`
- 课件所有题目必须是教材原文明确编号的题目，禁止 LLM 自行生成

### 13b. 例题选用规则

- 课件中的例题优先从 knowledge/workbooks/ 对应章节中选题
- 选题依据 习题分析skills 的8维度分析报告
- 每层学生在课件中至少应有 1 道选自习题库的对应层次题目
- 后台题源字段只写入课堂提问参考答案，不进入学生课件。
- 课件题目来源标注格式：`（来源：习题库·题号 / 教材例题）`
- 分层提问策划所涉及的习题库题目，必须与 学生成绩数据.md 的三层分组匹配

## 15. 图片引用规则

所有图片引用（来源、路径、格式、布局、验证）遵循全局统一规则，详见：

> **[图片资源skills](../../skills/images/SKILL.md)**

课件场景下特别说明：

- 课件中的图片引用路径统一为 `./images/文件名`
- `/courseware-collab` 使用教材或习题图片时，必须先将图片复制或登记到 `outputs/images/`
- 课件正文禁止出现非同级 `images/` 的图片路径
- 连续图片并列布局参照 图片资源skills §4
- 课件生成后须通过 **[图片资源validators](../../validators/images/rules.md)** 校验

## 16. 视频引用规则（可选）

视频引用是可选功能，仅在教师明确提到"引用视频""引入视频片段""使用视频微课"等表述时启用。

### 16.1 触发条件

仅当教师指令中包含以下表述时，才执行视频引用环节：
- "引用视频"
- "引入视频片段"
- "使用视频微课"
- 或其他明确表示需要视频的指令

### 16.2 视频引用标记格式

课件中使用 blockquote 格式标记视频插入点，放在对应活动页面标题下方：

```markdown
> 📹 视频片段：条形统计图概念引入
> - 来源：video_clips2/02_条形统计图.mp4
> - 播放时长：45秒
> - 播放时机：活动4开始时，作为条形图概念引入
> - 内容摘要：展示条形统计图的横轴、纵轴结构，说明柱高表示数量大小
> - 教学目的：帮助学生直观理解条形图表达数量信息的特点
> - 备注：视频详细讲解绘制方法，本课时仅需概念引入，跳过绘制步骤
> - 播放范围：00:25-01:10（跳过绘制步骤讲解）
```

### 16.3 必填字段

| 字段 | 说明 | 示例 |
|-----|------|------|
| 视频片段名称 | 视频片段的简短标识 | 条形统计图概念引入 |
| 来源 | 视频文件相对路径 | video_clips2/02_条形统计图.mp4 |
| 播放时长 | 建议播放时间长度 | 45秒 |
| 播放时机 | 在哪个活动环节、何时播放 | 活动4开始时，作为条形图概念引入 |
| 内容摘要 | 视频片段核心内容概述 | 展示条形统计图的横轴、纵轴结构 |
| 教学目的 | 播放此视频的教学意图 | 帮助学生直观理解条形图表达数量信息的特点 |

### 16.4 可选字段

| 字段 | 说明 | 示例 |
|-----|------|------|
| 备注 | 需要教师注意的事项 | 视频情境与教材情境不同，需做好过渡衔接 |
| 播放范围 | 视频具体播放时间范围 | 00:25-01:10（跳过绘制步骤讲解） |

### 16.5 视频引用位置

视频引用标记放在对应活动页面标题下方，作为教师备课提示信息，不作为课件正文内容呈现给学生。

### 16.6 视频时长控制

视频播放时长不应超过对应活动时长的一半，确保有足够时间进行师生交互。

### 16.7 验证要求

视频引用仅作为教师备课提示，无强制验证要求，但建议：
- 视频文件路径真实存在
- 视频时长与活动时长匹配
- 视频内容与教学活动目标一致

## 17. 课后作业规则

- 课后作业页标题 emoji 使用 `📝`
- 作业分为三个类别，**禁止使用"基础层/中间层/拓展层"等层级术语**：
  - **必做**：全体学生必须完成的基础巩固题
  - **选作**：有能力的学生可选做的提高题
  - **挑战**：供学有余力学生挑战的拓展题
- 每类作业数量：必做 2-3 题，选作 1-2 题，挑战 1 题
- **作业题目来源优先级**：
  1. 教材习题A组和B组中未被选为课堂练习/检测的剩余题目（自动进入课下作业池）
  2. 从 knowledge/workbooks/ 对应章节中补充选题
- 作业页格式示例：

```markdown
### 📝 课后作业

**必做**：
1. 教材 Pxx 练习第 1、2、3 题
2. 练习册 Pxx 第 4、5 题

**选作**：
1. 教材 Pxx 拓展练习第 6 题

**挑战**：
1. 思考：如何用今天学的方法解决生活中的实际问题？
```

- 禁止在作业页出现"层级""基础层""中间层""拓展层"等表述
- 禁止给作业题标注难度等级数字（如 ★★★）
- 课后作业中来自教材的题目必须标注教材页码和题号，来自练习册的题目必须标注练习册页码和题号

## 18. 问题先行规则（Question-First）

> 所有课堂提问按 **问题页 → 按需备用提示页 → 答案/证明/推理链页** 排列。禁止提示先于问题，禁止将教师参考预期写入学生课件。

### 18.1 规则层级

| 层级 | 规则内容 | 适用范围 |
|:---|:---|---:|
| 强约束 | 提问环节页必须在推理链分析页和规范证明页之前 | 所有复杂例题（需拆为4子页的） |
| 强约束 | 简单例题/判断题的课堂提问必须在答案页之前 | 所有含独立答案页的题目 |
| 建议性 | 小结页的三问三答按"基础→中间→拓展"顺序排列，不涉及先后问题 | 课堂小结页 |

### 18.2 正确顺序与错误顺序

**正确（Question-First）**：

题目展示 → **💬 问题引导（提问）** → 💡 备用提示1/2（按需） → ✏️ 推理链分析 → ✏️ 规范证明

**错误（Answer-First，禁止）**：

题目展示 → ✏️ 推理链分析 → ✏️ 规范证明 → 💬 问题引导（提问）

**错误（Hint-First，禁止）**：

题目展示 → 💡 备用提示 → 💬 问题引导（提问） → 答案

### 18.3 页面标题映射

| 子页类型 | 标题emoji | 标题关键词 |
|:---|:---:|:---|
| 题目页 | ✏️ | 例题X：xxx |
| 问题引导页 | 💬 | 问题引导 |
| 推理链分析页 | ✏️ | 推理链分析 |
| 规范证明页 | ✏️ | 规范证明 |

### 18.4 例外规则

以下场景允许提问在答案之后：

- **课堂出口检测**（本课小结的三问三答）：不涉及新例题解答，用于回顾而非引导
- **判断题/选择题的快速检测**（如"以下哪些是菱形判定"）：答案本身不剧透后续推理
- **第1页判定路径辨析的课堂提问**：这些是前置知识激活，不是某一例题的引导性提问

### 18.5 质检要求

课件生成后须逐页核对每个例题的页面顺序：

```markdown
质检流程：
1. 找到每一个以"### ✏️ 例题"开头的题目页
2. 检查其后的子页是否包含"### 💬 Xxx 问题引导"（提问页）
3. 确认提问页在推理链分析页和规范证明页之前
4. 如果提问页出现在推理链或证明之后 → 标记为违规 → 重新排序
```
