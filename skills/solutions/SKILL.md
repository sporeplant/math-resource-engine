# 教材问题解答

按教材原文顺序，为教材中要求学生回答、判断、计算、证明、操作、作图、交流或表达的全部任务生成可追溯参考解答。

## 强制读取

1. 读取仓库根目录 `AGENTS.md`。
2. 读取 `orchestrator/skill-protocol.md` 和 `orchestrator/workflow-registry.md`。
3. 读取本目录 `spec.md` 和 `检查清单.md`。
4. 读取目标课时对应的完整教材原文及其图片。
5. 读取 `validators/solutions/rules.md`。
6. 读取 `knowledge/chapter-knowledge-points.yaml` 获取本章核心关键词。

## 执行流程

1. 匹配唯一教材课时，不确定时先完成课题确认。
2. 按原文顺序提取全部显式问题和任务指令，拆分大题与小问，绑定相邻图片。
3. 先形成任务清单，核对栏目、顺序、数量、小问和图片依赖。
4. 判断教材是否已经给出解答：使用 `答案来源: 教材原文` 或 `答案来源: AI参考推导`。
5. 按题型模板逐题作答，不改写原题，不改变教材顺序，不补造题目。
6. 对计算、推理、证明、图形和结论进行复核。
7. 运行 `python tools/check_solution_knowledge.py --chapter {章号}` 检查知识点一致性，复核未命中题目并修正真问题。
8. 执行检查清单与专用validators，通过后写入规定目录。
9. 执行例题-习题全链台阶分析，输出升格建议，追加到解答文件末尾。

## 例题-习题全链台阶分析

在全部解答完成并写入后执行。目的是判断 B/C 组习题与例题之间是否存在方法断崖，输出升格建议供活动设计参考。

### 分析范围

仅覆盖教材内部：例题 → 做一做 → 练习 → A组 → B组 → C组。不分析练习册题目。

### 方法标签

为每道题标注方法标签，标签从 `knowledge/chapter-knowledge-points.yaml` 中提取本章核心关键词推导，控制在 5-8 个/章节。标签格式：`方法名:细分`，如 `去分母:单因式`、`性质:等量代换`。

### 全链扫描逻辑

从例题开始，按栏目顺序逐题扫描，维护"已出现方法集合"：

1. 例题的方法标签自动进入集合。
2. 后续每道题：检查其方法标签是否已在集合中。
3. 不在集合中 → 记录为"首次出现"，同时检查前序栏目是否已引入（作为教材铺设的台阶）。
4. 首次出现且前序无台阶 → 标记为"方法断崖"。

### 升格判断

仅当 B/C 组中出现方法断崖时触发：

- 升格数 ≤ 1，最终例题总数 ≤ 2。
- 升格建议 = 从断崖题中选取 gap 最大的 1 道。
- 若多道题存在断崖且同类型，选最前面的那道，其余后移习题课。
- 若多个不同类型的断崖，选对本课核心目标影响最大的那道。

### 不升格条件

以下情况即使 B/C 组有新方法也不升格：

- 新方法已在中间栏目（做一做/练习/A组）引入。
- 新方法是其他章节已学内容，不属于本课新知。
- 课时时间不允许（已接近 40 分钟上限）。

### 建议性质声明

升格建议仅为分析结果，不自动改写教学设计。活动设计 skills 结合课时时间和学情做最终决策，由教师在确认门 4-1 确认是否采纳。

## 异常处理

正常题目不写审核状态或生成状态。仅当题目残缺、图片缺失或条件无法可靠辨认时写：

```yaml
生成状态: 暂停
异常原因: "具体原因"
```

禁止猜测缺失条件或根据常见题型补题。
