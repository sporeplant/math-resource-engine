# 课堂提问调度稿生成skills

## 1. 定位

- 本skills是 **课件设计skills** 的内置子环节，在分层提问skills完成学生选取与问题-学生映射后执行。
- 独立生成 `{课时名}_课堂提问调度稿.md` 文件，不与学生课件生成混杂。
- outputs供教师课堂使用的完整提问调度信息、备用提示、即时调整规则、参考答案和评分标准。
- 学生课件不显示具体学生姓名；具体点名、备用学生和调整策略只进入教师调度稿。

---

## 2. 输入要求

| 输入项 | 来源 | 要求 |
|--------|------|------|
| 教学设计文件 | `outputs/{课时名}_教学设计.md` | `review_status: 审核通过` |
| 课件文件 | `outputs/{课时名}_课件.md` | 学生投屏正文，不含具体学生姓名 |
| 学生选取结果 | 分层提问skills outputs | 包含随机池选取记录和问题-学生映射表 |
| 题源文件 | `knowledge/workbooks/`、`knowledge/textbooks/` | 所有引用的题目完整内容 |
| 教材参考解答 | `knowledge/solutions/ch{章节号}/solution-{lesson_id}.md` | 必须存在、课时匹配并通过教材问题解答validators |
| 练习册答案 | `knowledge/workbook-answers/` | 所有练习册题目的参考答案来源 |
| 练习册逐题索引 | `knowledge/workbook-index/` | `source_type: exercise_bank` 题目的 `question_id` 和答案映射 |
| 习题分析报告 | 习题分析skills outputs | 8维度分析结果 |

---

## 3. outputs结构

### 3.1 文件元数据

```yaml
---
content_type: question_dispatch
lesson_id: "LESSON-XX.X-XX"
lesson_name: "XX.X第X课时"
command: courseware-collab
workflow_version: v2
review_status: draft
source_files:
  - "outputs/{课时名}_教学设计.md"
  - "outputs/{课时名}_课件.md"
  - "knowledge/solutions/ch{章节号}/solution-{lesson_id}.md"
created_at: "YYYY-MM-DD"
collab_gates:
  structure: confirmed
  questions: confirmed
  assignment: confirmed

```

## 4. 板块A生成规则

### 4.0 教材答案基线

- 若 ASK 问题直接对应教材任务，必须按 `question_id` 读取教材参考解答，以其中的数学结论、解题依据和合理答案边界作为事实基线。
- 可将教材参考解答整理为适合课堂提问的 3-5 个要点和评分点，但不得改变结论、补造条件或超出教材认知边界。
- 必须保留原条目的 `答案来源` 属性；`AI参考推导` 和 `AI参考推导补充` 不得表述为教材原文或出版社标准答案。
- ASK 问题不直接对应教材任务时，仍按教学设计和真实题源生成，不得伪造教材 `question_id`。

### 4.1 问题对应关系

- 从学生课件和教学设计中提取所有课堂提问问题。
- 每个问题必须对应：
  - 问题编号（如 ASK-B-01）
  - 课件页码或页面标题
  - 所属活动（如 ACT-B-01）
  - 建议提问学生（来自分层提问skills的选取结果）
  - 备用学生（可选，用于答不出或临时调整）
  - 问题原文（必须与学生课件一致）
  - 提问意图与观察证据
  - 分级备用提示（至少1级，注明仅在学生停滞或答偏时使用）
  - 参考答案（3-5个要点）
  - 即时调整规则
  - 评分要点（2-4个评分项，每项配分值）

### 4.2 调度质量要求

| 要求 | 说明 |
|------|------|
| **隐去学生端姓名** | 具体姓名只进入调度稿，不进入学生课件 |
| **分层适配** | 基础层答案侧重事实性，中间层侧重推理性，拓展层侧重开放性 |
| **可现场执行** | 每个问题都有可用提示、追问或备用学生策略 |
| **观察可见** | 提问意图必须对应学生可观察行为或回答证据 |
| **数学准确性** | 所有数学公式、定理、推理步骤必须准确无误 |

### 4.3 即时调整规则

- 若学生答不出：先给最小支架，不直接公布答案；必要时请备用学生补充，再让原学生复述关键点。
- 若学生答偏：指出偏差所在的问题对象或条件，引导回到题干数据、图表或定义。
- 若学生答出基本点：追加“为什么”“依据是什么”“还能怎样表示”等理由追问。
- 若学生答得完整：追加比较、评价、迁移或反例辨析追问。

---

## 5. 板块B生成规则

### 5.1 覆盖范围

必须覆盖教学设计 `lesson_flow` 和学生课件中引用的：

- 所有教材例题。
- 所有课堂练习/检测题目。
- 所有习题库题目。
- 台阶递进式核心题（含完整解题步骤，不仅是最终答案）。

教材例题、教材练习和教材习题的解答必须按 `question_id` 从教材参考解答中复用，不得再次独立推导。允许为页面长度压缩表达，但必须保留关键步骤、数学结论、解题依据和 `答案来源`。练习册题目必须按 `question_id` 从练习册逐题索引定位答案文件，不得脱离答案册自行推导。

### 5.2 题源对应要求

每个题目必须标注：

- `source_id`（如 TEXTBOOK-22.1-01）
- `source_type`（textbook / exercise_bank）
- `question_id`（如 Q-22.1-05 / WB-12.1-1-Q001-S01）

### 5.3 解答质量要求

| 题型 | 解答要求 |
|------|---------|
| **证明题** | 完整推理链，每步有依据 |
| **计算题** | 关键计算步骤，最终结果 |
| **概念题** | 准确定义 + 辨析说明 |
| **应用题** | 建模过程 + 求解步骤 + 结论解释 |

---

## 6. 页面长度控制

- **提问对象建议表**：尽量控制在 1 页 A4 纸内。
- **板块A**：按课件页码顺序排列，便于教师现场查找。
- **板块B**：作为附录，允许略长，但不得省略关键步骤。
- 若题量较大，优先保证课堂提问调度信息完整，例题与练习解答可作为附录分节呈现。

---

## 7. 数学准确性要求

- 禁止出现数学概念错误。
- 禁止出现公式、定理表述错误。
- 禁止推理逻辑断裂。
- 禁止计算错误。
- 所有数学符号使用 LaTeX 格式。
- 几何证明步骤符合规范。
- 计算过程展示关键中间结果。

---

## 8. 与分层提问skills和课件设计skills的协作流程

```text
分层提问skills
    ↓ outputs：学生选取结果 + 问题-学生映射表
课堂提问调度稿生成skills
    ↓ 输入：教学设计 + 学生课件 + 学生选取结果 + 题源
    ↓ outputs：课堂提问调度稿文件
课件设计skills
    ↓ 验证：学生课件不显示学生姓名；调度稿姓名与学生选取结果一致
```

---

## 9. 禁止事项

- 禁止使用 LLM 自行生成题目（所有题目必须来自题源）。
- 禁止省略板块A或板块B。
- 禁止答案与问题不匹配。
- 禁止评分要点模糊不清。
- 禁止数学错误。
- 禁止在学生课件中写入具体学生姓名。
- 禁止在教材参考解答缺失、课时不匹配、验证失败或缺少所需 `question_id` 时继续任务。
- 禁止自动补齐教材参考解答、降级为临时推导或跳过校验。
- 禁止重新推导教材题后覆盖教材参考解答中的结论和答案来源。
- 禁止练习册题目缺少逐题索引或答案映射时继续生成调度稿。
