# 分层提问skills

## 1. 定位

- 本skills是 **课件设计skills** 的内置子环节，在参考答案生成skills之前执行
- 独立完成学生姓名分配，outputs学生选取结果供参考答案生成skills使用
- 不负责生成参考答案，仅负责"问谁"

---

## 2. 分层标准

以 `students/scores.md` 中的学生最近一次考试分数为依据：

| 分层 | 子层 | 分数范围 | 可回答的问题类型 |
|:---|:---|:---:|:---|
| **基础层** | 基础A | 0-24分 | 记忆/回忆类（背公式、读图、复述概念、选择判断） |
| | 基础B | 25-49分 | 理解/简单应用类（解释推导、简单计算、模仿步骤） |
| **中间层** | — | 50-69分 | 推理应用类（思路解释、方法迁移、逻辑分析、书写证明） |
| **拓展层** | — | 70-100分 | 拓展探究类（本质分析、特点归纳、变式创新、思想提炼） |

**核心约束**：
- 基础A学生 **只能** 分配记忆/回忆类问题
- 基础B学生可分配基础层全部问题（记忆+理解）
- 问题设计时，必须在教学设计中标注问题认知难度，以便随机池按子层匹配

---

## 3. 学生姓名选取规则

### 3.1 通用化原则

- 教学设计docs中**不得写死**具体学生姓名
- 只标注提问所属层级，如：`请一位基础层同学回答`
- 具体学生姓名在**最终生成课件时**，由本skill的随机池算法动态填入

### 3.2 数据源

- 仅从 `students/scores.md` 中读取四池名单（基础A / 基础B / 中间层 / 拓展层）
- 分层对应规则：
  - 记忆/回忆类问题 → 基础A 或 基础B 池
  - 理解/简单应用类问题 → 基础B 池
  - 推理应用类问题 → 中间层 池
  - 拓展探究类问题 → 拓展层 池
- 基础A学生不得分配理解类及以上问题

### 3.3 完整执行流程

```
输入：
  - 教学设计文件（含各层问题数：k_basicA, k_basicB, k_mid, k_ext）
  - students/scores.md（四池独立 + 教师标记）
  - students/ask-log.json（跨课时历史）

outputs：
  - 选中的学生名单（四池独立）
  - 更新后的提问历史记录.json

步骤：
1. 从教学设计文件中统计：
   - k_basicA = 记忆/回忆类基础问题数
   - k_basicB = 理解/简单应用类基础问题数
   - k_mid = 中间层问题数
   - k_ext = 拓展层问题数

2. 四池独立处理（每个子层执行相同步骤）：
   a. 从学生成绩数据加载该层原始池
   b. 从历史记录中读取该层学生的历史提问次数
   c. 从学生成绩数据中读取该层学生的教师标记：
      - "skip" → 从池移除
      - "priority" → 权重=2.0
   d. 应用历史权重：
      - 连续2课未被提问 → 权重=1.5
      - 普通 → 权重=1.0
   e. 执行加权 Fisher-Yates，抽取 k 人
   f. 把选中的学生写入历史记录的该层 picks 数组

3. 返回四池选中名单
```

### 3.4 约束

- **分层独立**：四个子层（基础A/基础B/中间/拓展）完全独立，各自执行加权 Fisher-Yates
- **不重复**：同一课时内，每位学生最多被提问一次
- **全覆盖倾向**：若某层提问人次接近该层人数，应尽量覆盖该层所有人
- **跨子层禁止**：
  - 记忆/回忆类问题：可从基础A和基础B池抽取
  - 理解/简单应用类问题：只能从基础B池抽取，禁止从基础A池抽取
  - 中间层以上问题：严禁从基础层（A/B）抽取
- **k = 该层问题数**：每一层抽取人数严格等于该层问题总数

## 4. 加权 Fisher-Yates 洗牌算法

### 4.1 与标准版本的区别

| 特性 | 标准 Fisher-Yates | 加权 Fisher-Yates |
|:---|:---|:---|
| 每个元素权重 | 均为 1 | 可自定义 |
| 抽取概率 | 等概率 | 与权重成正比 |
| 适用场景 | 单课时随机提问 | 多课时间隔均衡 + 教师干预 |

### 4.2 伪代码

```
输入：
  - pool: [ (name1, weight1), (name2, weight2), ... ]
  - k: 需要抽取的人数
outputs：
  - selected: 选中的 k 个姓名（不重复）

步骤：
1. 计算总权重 total_weight = sum(weight_i)
2. 复制一份工作池 work_pool = pool.copy()
3. 初始化 selected = []
4. 重复 k 次：
   a. 在 [0, total_weight) 区间内取随机数 r
   b. 累加 work_pool 元素的权重，直到累加和 ≥ r，找到对应的元素
   c. 把该元素加入 selected
   d. 从 work_pool 中移除该元素，从 total_weight 中减去其权重
5. 返回 selected
```

## 5. 教师标记与历史权重

### 5.1 教师标记说明

教师可在 `students/scores.md` 的"教师标记"列中为每位学生手动标注：

| 标记 | 效果 | 权重 |
|:---|:---|:---:|
| `skip` | 从池移除，不参与抽取 | 0 |
| `priority` | 优先抽取 | 2.0 |
| 空 | 普通 | 见下文 |

### 5.2 历史权重说明

| 学生历史状态 | 权重 |
|:---|:---:|
| 普通学生（最近3课被提问 ≤ 1次） | 1.0 |
| 连续3课未被提问 | 1.5 |

### 5.3 标记示例

在学生成绩数据中标注后，读取结果示例（算法内部构造的加权池）：

```
基础层工作池（读取自学生成绩数据.md）：
  陈虹羽: weight=1.5（连续2课未提问）
  韩亚彤: weight=1.0（普通）
  王澎辉: weight=2.0（priority标记）
  谢昕达: 已移除（skip标记）
```

## 10. 问题链的台阶递进与课堂提问

### 10.1 核心理念

```
低起点台阶① → 问题① → 台阶② → 问题② → 台阶③ → 问题③ → 核心结论
```

每级"台阶+问题"形成一个微循环：

1. **台阶**：教师将问题拆解一小步（给提示、化条件、引方向），不是给完整解答
2. **问题**：学生在当前台阶上思考并回答，推进解决进程
3. **反馈**：即时确认，再进入下一台阶

---

### 10.2 分层嵌入规则

| 台阶位 | 问题难度 | 适合回答层 | 问题类型 |
|:---|:---:|:---|:---|
| 第①问（起点台阶） | 低 | 基础层 | 观察、辨认、回忆 |
| 第②问（推进台阶） | 中 | 中间层 | 分析、推理、解释 |
| 第③问（收束台阶） | 高 | 拓展层 | 归纳、评价、迁移 |

- 同一道题内的递进问题形成一条**问题链**，各层学生各答一段
- 基础层答第①问即有参与感，中间层答第②问推进核心推理，拓展层答第③问完成收束
- 各问之间由问题解决的逻辑自然连接，不需要额外的过渡语

---

### 10.3 课堂示例

以 A10 正方形线段和问题为例，问题链如下：

```
台阶①：连接PO，图中出现了哪几个三角形？ → 基础层可答 △PCO、△PDO、△CDO
台阶②：△PCO和△PDO的面积分别怎么算？试着用PE、PF写出来。 → 中间层可答
台阶③：除了面积法，还有其他方法证明PE+PF等于半条对角线吗？ → 拓展层可答
```

三个问题自然形成一条链：**看见图形 → 写出表达式 → 迁移方法**，每一步在前一步基础上推进，不跳跃。

---

### 10.4 注意事项

- 台阶①必须足够低，确保基础层学生能参与（所有学生都有事可做）
- 台阶之间的跨度不宜过大，中间层学生应能独立完成跳跃
- 如果学生在某台阶卡住，教师应追加更小的中间台阶，而非直接给出答案
- 问题链的总台阶数控制在 3-4 级，超过则认知负荷超载

---

## 11. 注意事项

1. **禁止虚构学生姓名**：所有学生姓名必须从成绩数据文件中提取
2. **禁止跨子层提问**：基础A学生不回答理解类问题，基础层学生不回答中间层以上问题
3. **教学设计中不写死姓名**：教学设计阶段只标注层级与认知难度，不填入具体学生姓名
4. **课件生成时动态填入**：具体姓名在课件生成阶段，由随机池算法从数据源读取并填入
5. **分层不是标签**：课堂用语中不要给学生贴标签，分层仅作为教师提问依据
6. **灵活调整**：若某学生多次提问表现优异，下次课时可适当提升其所属层次
7. **重复提问策略**：若某层人数不足，应优先从该层已有学生中轮换，而非跨层调用
8. **基础A低分保护**：基础A学生（0-24分）只分配记忆/回忆类问题，避免因认知负荷过高而挫伤学习信心
