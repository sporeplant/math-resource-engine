# 一致性验证器

## 验证项

- [ ] 学习目标与评价是否一致（每条目标都有对应的评价任务）
- [ ] 评价与活动是否一致（评价方式能与教学活动匹配）
- [ ] 活动是否支撑目标（活动内容直接服务于目标达成）
- [ ] 是否存在无效活动（活动与目标无关，或可有可无）
- [ ] 是否存在无法评价目标（有目标但没有对应的评价设计）
- [ ] 是否符合逆向设计（目标 → 评价 → 活动的设计顺序）

---

## 验证流程

1. 对每条学习目标，找到对应的评价任务
2. 对每个评价任务，找到对应的教学活动
3. 检查是否存在无对应评价的"空目标"
4. 检查是否存在不支撑目标的"无效活动"
5. 检查整体的目标-评价-活动链条是否完整
6. 确认设计顺序符合逆向设计逻辑

---

## 验证结果

- 通过：目标-评价-活动链条完整一致
- 有条件通过：存在链条断裂，附修复建议
- 不通过：严重不一致，需退回重新设计
