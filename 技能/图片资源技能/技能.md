# 图片资源技能

定义图片引用的全局规则：来源、路径、格式、布局、验证。

> 本技能为全局规则提供者，不独立执行，被课件设计技能、教材文件、习题文件等多处引用。

---

## 1. 图片来源规则

- 每个需配图的例题、练习、拓展挑战，必须在题目下方引用对应图片。
- 严禁猜测或编造图片文件名。图片文件多为哈希命名，无法推测。
- 获取正确图片文件名的唯一方式：读取源 Markdown 文件，从其中复制图片文件名。
- `/courseware` 使用知识库图片时，必须将图片复制或登记到 `输出/images/`，课件正文只引用 `./images/文件名`。

---

## 2. 图片路径规则

任意 Markdown 文件引用图片时，只允许引用该文件同级目录下的 `images/` 子目录。

| 引用文件所在位置 | 图片位置 | 路径写法 |
|---|---|---|
| `知识库/教材原文/{章节}.md` | `知识库/教材原文/images/` | `./images/文件名` |
| `知识库/练习册题库/{章节}.md` | `知识库/练习册题库/images/` | `./images/文件名` |
| `输出/{课时名}_完整教学设计.md` | `输出/images/` | `./images/文件名` |
| `输出/{课时名}_课件.md` | `输出/images/` | `./images/文件名` |
| `输出/{课时名}_课堂提问参考答案.md` | `输出/images/` | `./images/文件名` |

路径必须指向真实存在的文件，生成后需通过图片资源验证器校验。

---

## 3. 图片格式规则

- 使用 HTML `<img>` 标签：`<img src="./images/文件名" width="35%">`
- 禁止使用 Markdown `![]()` 语法。
- 默认 `width="35%"`，统计图表等确需放大时可用 `width="45%"`，并在图片后说明原因。
- 图片引用独占一行，放在题目描述之后、分析思路或提示之前。

---

## 4. 连续图片并列规则

当出现 2 张及以上连续图片，且图片之间只有图注、编号等短文本时，必须用 Markdown 表格并列显示。

标准模板：

```markdown
| 图注1 | 图注2 |
|:---:|:---:|
| <img src="./images/a.jpg" width="35%"> | <img src="./images/b.jpg" width="35%"> |
```

表格规则：

- 2 张：1 行 2 列。
- 3 张：2 行，尾行左图右空。
- 4 张：2 行 2 列。
- 5 张及以上：优先每行 2 列。
- 表格内不得使用额外空行增加高度。

---

## 5. 占位符规则

无法确认图片文件名时，草稿中可使用：

```text
[图片待确认：请从源教材或习题 Markdown 的 ./images/ 目录核对]
```

- 占位符不得在最终输出中保留。
- 图片资源验证器必须拦截最终输出中的占位符。

---

## 6. 引用来源标注

- 课件中的配图需在图片下方或表格下方标注图片所属章节。
- 教材图片标注：`（图源：教材 第X章）`
- 习题图片标注：`（图源：习题库·{章节}）`

---

## 7. 自检清单

- [ ] 所有图片是否使用 `<img>` 标签。
- [ ] 所有图片路径是否为 `./images/文件名`。
- [ ] 是否无跨目录图片路径和盘符绝对路径。
- [ ] `<img>` 的 `width` 是否为 `35%` 或有说明的 `45%`。
- [ ] 连续图片是否已用表格并列。
- [ ] 图片文件是否真实存在。
- [ ] 最终输出是否无图片占位符。
