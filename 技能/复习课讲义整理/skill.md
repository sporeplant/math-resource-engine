# 复习课讲义整理技能

将多份新授课讲义合并整理为 1 份综合复习讲义。

---

## 一、输入与输出

**输入**：由用户在调用时指定（讲义编号或名称），不做任何默认假设。

**输出路径**：`输出/八下复习讲义/`（固定）。

**输出文件名**：由用户在调用时指定。

---

## 二、三模块结构

| 模块 | 内容 | 数量 | 编号方式 |
|------|------|------|----------|
| 1. 知识点与例题 | 源讲义的全部知识点，每个配 1 道源题 | 与知识点数一致 | 1．2．3．... |
| 2. 当堂练习 | 全部 题型N 典例（全量纳入） | 由源文档决定 | 接续模块 1 |
| 3. 课后作业 | 从变式中选取 10 题 | 固定 10 题 | 接续模块 2 |

**横向规则**：全讲义一套顺序编号，不重置。

**均衡规则**：知识点按源讲义顺序排列，不合并、不打乱。全部知识点和全部题型N 典例都纳入整理后文档，确保源文档内容完整覆盖。

**比例规则**：题型和难度尽量贴近源讲义原文比例，保留基础题、中等题、中等偏难题的梯度，不得全部集中在同一题型或同一难度。

**源文档结构识别规则**：源讲义通常先列出知识点与「即学即练」，再进入「题型精讲」区域。题型精讲区域中：

- `## 题型 NN ...` 表示一个题型板块。
- `【典例 1】`、`【典例1】` 等表示该题型板块的典型例题，全部纳入当堂练习。
- `【变式 1】`、`【变式1】` 等表示该题型板块的例题变式，是课后作业的选题来源。
- 变式题默认归属于其前最近的 `## 题型 NN ...` 和对应典例，不得脱离原题型重新归类。
- 题型精讲区域之后连续出现的 `1．`、`2．` 等编号题，视为综合练习或课后练习来源，优先级低于典型例题和例题变式。

---

## 三、生成前：源文件统计与确认

在正式生成讲义前，必须先对各源文件进行统计，并呈现给用户确认。

### 统计内容

对每个源文件分别统计：

| 统计项 | 说明 |
|--------|------|
| 知识点数量 | 源文件中 `## 知识点` 的数量 |
| 即学即练题数 | 源文件中「即学即练」区域的总题数 |
| 题型N 典例数 | 源文件中各 `## 题型 NN` 板块下 `【典例...】` 的数量（按题型板块分别列出） |
| 题型N 变式数 | 源文件中各 `## 题型 NN` 板块下 `【变式...】` 的数量（按题型板块分别列出） |

### 呈现格式

```markdown
📊 源文件统计：

| 源文件 | 知识点数 | 即学即练题数 | 题型N 典例数 | 题型N 变式数 |
|--------|---------|-------------|-------------|-------------|
| 03讲   | 3       | 3           | 题型01: 2, 题型02: 1, 题型03: 2 | 题型01: 2, 题型02: 2, 题型03: 1 |
| 04讲   | 2       | 2           | 题型04: 1, 题型05: 2 | 题型04: 3, 题型05: 2 |

汇总：
- 知识点总数：5
- 即学即练总数：5
- 题型N 典例总数：8（全部纳入当堂练习）
- 题型N 变式总数：10（课后作业选题池）

请确认以上统计。课后作业固定选用 10 题，确认后继续生成。
```

### 确认流程

1. 呈现统计表格
2. 用户确认统计数据；课后作业数量固定为 10 题
3. 用户确认后，锁定统计数据，继续生成

---

## 四、模块 1：知识点与例题

### 知识点选取

- 不合并，源讲义有几个知识点就保留几个。
- 完整保留每个知识点的概念讲解文字。

### 例题选取

- 来源：优先从每个知识点对应的「即学即练」选取 1 道；若该知识点没有合适的「即学即练」，可从同知识点的「题型精讲」典例或强化训练中选取。
- 数量：与知识点数量一致。
- 对应关系按源文档顺序判断：每个知识点优先匹配该知识点标题之后、下一个知识点或题型精讲区域之前最近的「即学即练」，不得按 `【即学即练1】`、`【即学即练2】` 等编号做全局匹配。
- 若例题提取后只剩题号（如 `4．`）或题号后无题干，必须回退重新提取，不得输出空题干。

### 格式

```markdown
## 知识点{双位编号} 标题

知识点概念讲解文字（完整陈述，不留空）……

## 例题讲解

1．题目文字……

（1）第1小问

（2）第2小问

![](images/{哈希值}.扩展名)
```

---

## 五、模块 2：当堂练习

### 选题规则

当堂练习纳入源文档中**全部** 题型N 典例，不进行筛选。

- 以题型精讲区域内的 `【典例...】` 标记为准，按题型板块顺序逐个收录。
- 每个典例保留其所在题型板块信息。
- 源文档中所有题型N 的典例务必全部纳入，不遗漏。

### 数量

由源文档决定，不设固定上限或下限。若源文档无题型N 典例，则当堂练习为空。

### 编号

接续例题编号（如例题到第 3 题，当堂练习从第 4 题起）。

---

## 六、模块 3：课后作业

### 选题规则

课后作业从变式中固定选取 10 题。

优先级：① 源文档中的例题变式；② 未入选例题/当堂练习的其他变式题；③ 有图的综合题；④ 原课后练习综合题。

例题变式必须优先用于课后作业，用来承接课堂典例并形成迁移训练。按变式题所属题型板块均衡选取，保持"典例-变式"的训练关系。

识别例题变式时，以题型精讲区域内的 `【变式...】` 标记为准；变式题应优先跟随其所属题型和对应典例进入课后作业。

### 数量

固定选取 10 题。若变式池不足 10 题，可按优先级补充有图综合题或原课后练习综合题；若全部来源题不足 10 题，必须在生成前报告并停止，不得虚构题目。

### 编号

接续当堂练习，全讲义一套顺序编号，不重置。

---

## 七、文档末尾：原始数量与选用数量对比表

在讲义正文末尾，附一份原始数量与选用数量的对比表格。

### 格式

```markdown
## 原始数量与选用数量对比

| 来源 | 类别 | 原始数量 | 选用数量 | 备注 |
|------|------|---------|---------|------|
| 03讲 | 知识点 | 3 | 3 | 全部保留 |
| 03讲 | 即学即练 | 3 | 3 | 全部用作例题 |
| 03讲 | 题型01 典例 | 2 | 2 | 全部纳入当堂练习 |
| 03讲 | 题型01 变式 | 2 | 1 | 课后作业选用 |
| 03讲 | 题型02 典例 | 1 | 1 | 全部纳入当堂练习 |
| 03讲 | 题型02 变式 | 2 | 1 | 课后作业选用 |
| 04讲 | 知识点 | 2 | 2 | 全部保留 |
| 04讲 | 即学即练 | 2 | 2 | 全部用作例题 |
| 04讲 | 题型04 典例 | 1 | 1 | 全部纳入当堂练习 |
| 04讲 | 题型04 变式 | 3 | 2 | 课后作业选用 |
| 04讲 | 题型05 典例 | 2 | 2 | 全部纳入当堂练习 |
| 04讲 | 题型05 变式 | 2 | 1 | 课后作业选用 |
| **合计** | — | **24** | **21** | — |
```

### 字段说明

| 字段 | 说明 |
|------|------|
| 来源 | 源文件名称（如 03讲、04讲） |
| 类别 | 知识点 / 即学即练 / 题型NN 典例 / 题型NN 变式 |
| 原始数量 | 源文件中该类别下的实际数量 |
| 选用数量 | 整理后讲义中实际选用的数量 |
| 备注 | 简要说明选用规则（如"全部保留""全部纳入当堂练习""课后作业选用"） |

### 汇总行

- 最后一行「合计」统计原始总量和选用总量
- 原始数量合计 = 知识点数 + 即学即练题数 + 全部典例数 + 全部变式数
- 选用数量合计 = 知识点数 + 例题数 + 当堂练习题数 + 课后作业题数

## 八、格式细则

### 讲标题

```markdown
## 第 XX 讲 标题
```

- 使用 `##`（二级标题，不得使用一级标题 `#`）
- 格式固定：`## 第 XX 讲 内容标题`
- XX 为双位数字（01、02、14...）
- 标题后**不加任何元数据**（不加学习目标、思维导图等）

### 知识点标题

```markdown
## 知识点01 有序数对
```

- 使用 `##` 二级标题
- 格式：`## 知识点{双位编号} {知识点名称}`
- 双位编号顺序：01、02、03……
- 编号与标题之间有一个空格

### 概念讲解

- 普通段落文字，**不加粗、不加列表标记**
- 复习课中概念为**完整陈述**，不留空
- 行末使用中文句号 `。` 或全角句点 `．`

### 例题讲解标题

```markdown
## 例题讲解
```

- 固定写法 `## 例题讲解`，使用二级标题
- 每个知识点后紧跟一个"例题讲解"

### 题号格式

- 题号：`序号．`（**阿拉伯数字 + 全角句号**），如 `1．`、`2．`、`3．`
- 每个题号前有一个空行
- 小题标号：`（1）`、`（2）`、`（3）`
- 题目为普通段落文字
- 配图紧跟题后，图片前后各有一个空行

### 图片引用

```markdown
![](images/{哈希值}.扩展名)
```

- 格式：`![](images/{哈希值}.扩展名)`
- 路径：**相对路径** `images/`，与 .md 文件同目录
- 文件名：长哈希值（SHA256风格）
- 扩展名：`.jpg` 或 `.png`
- 图片前后各有一个空行

### 选择题选项

```markdown
A．$\frac{25}{4}$

B．$\frac{24}{4}$

C．2 或 $\frac{25}{4}$

D．2 或 $\frac{25}{4}$
```

- 选项字母 + 全角句号：`A．`、`B．`、`C．`、`D．`
- 每个选项**独立成行**
- 选项之间有一个空行
- 选项与题干之间有一个空行

### 数学公式

```markdown
行内：$a ^ { 2 } + b ^ { 2 } = c ^ { 2 }$
分数：$\frac { 2 5 } { 4 }$
角度：$9 0 ^ { \circ }$
三角形：$\triangle A B C$
下标：$_ { A B }$
```

- 统一使用 `$...$` 行内公式（不用 `$$` 块级公式）
- LaTeX 语法，字母和符号之间可以有空格
- 分数：`\frac {分子} {分母}`
- 角度：`\circ`
- 三角形：`\triangle`
- 下标：`_ { }`

### 空行规则

| 位置 | 空行数 |
|------|--------|
| 讲标题之后 | 1 个 |
| 知识点标题与概念讲解之间 | 1 个 |
| 概念讲解与例题讲解标题之间 | 1 个 |
| 例题讲解标题与第1题之间 | 1 个 |
| 例题之间 | 1 个 |
| 例题与配图之间 | 1 个 |
| 配图与下一模块之间 | 1 个 |
| 选择题选项之间 | 1 个 |
| 当堂练习/课后作业标题与第1题之间 | 1 个 |

## 九、配图搬运规则

### 搬运范围

只搬运被选用题目（模块 1/2/3 中实际引用的题目）引用的图片，不搬运整份源 `images/` 目录。

### 搬运流程

1. 识别所有被选用题目中的 `![](images/{哈希}.扩展名)` 引用。
2. 将对应的图片文件从源讲义 `images/` 目录复制到 `输出/八下复习讲义/images/`。
3. 输出讲义中的引用路径保持 `![](images/{哈希}.扩展名)` 不变，指向输出目录下的新图片。

### 示例

```
源：03讲.md → 引用 images/a1b2c3.jpg
操作：复制 a1b2c3.jpg → 输出/八下复习讲义/images/a1b2c3.jpg
讲义中引用：![](images/a1b2c3.jpg)（路径不变，自然指向输出目录下的新图片）
```

## 十、禁止事项

- 禁止修改源文件（知识库中的讲义）。
- 禁止虚构题目或改编题目。
- 禁止不按规则选取例题/当堂练习/课后作业。
- 禁止引用非同级 `images/` 的图片路径。
- 禁止只偏向某一个知识点或题型板块选题。
- 禁止遗漏任何题型N 典例（当堂练习必须全量纳入）。
- 禁止原始数量与选用数量对比表行数与统计项不一致。
- 禁止被选用题目的图片未搬运到输出文件同级 `images/` 目录。
- 禁止在统计确认前直接开始生成讲义。

## 十一、后续：DOCX 排版导出

复习讲义 md 文件生成后，如需打印分发：
1. 先运行验证：`python 技能/复习课讲义整理/validate_review_lesson.py 输出/八下复习讲义/{文件名}.md`
2. 通用 MD→DOCX（单栏）：`python -c "import sys; sys.path.insert(0,'tools'); from md2docx import process; process('{文件名}.md','{文件名}.docx')"`
3. 复习专用双栏压缩：`python tools/compact_review_docx.py {文件名}.docx --output {文件名}.docx`

详见 `技能/docx排版导出/skill.md`。

## 十二、工具脚本位置与标准调用

以下脚本为复习讲义整理、验证和导出时优先使用的本地工具。

| 用途 | 脚本位置 | 说明 |
|------|----------|------|
| 复习讲义整理辅助 | `tools/review_lesson_processor.py` | 历史辅助脚本；正式生成仍以本技能规则为准，使用前需核对是否符合当前三模块结构和确认流程 |
| 复习讲义专用验证 | `技能/复习课讲义整理/validate_review_lesson.py` | 生成 Markdown 后必须运行，检查 YAML、题量、连续编号、图片路径、对比表等 |
| 通用输出验证 | `tools/validate_output.py` | 专用验证通过后继续运行，检查全系统硬规则 |
| MD→DOCX 通用转换 | `tools/md2docx.py` | 系统级 MD→DOCX 转换（单栏，Pandoc），`from md2docx import process` |
| DOCX 命令行入口（轻量） | `tools/md_to_docx.py` | 轻量 MD→DOCX（纯 XML，无 Pandoc 依赖） |
| 复习讲义双栏压缩 | `tools/compact_review_docx.py` | 复习讲义专用后处理，双栏 + 压缩 |

标准验证命令：

```bash
python 技能/复习课讲义整理/validate_review_lesson.py 输出/八下复习讲义/{文件名}.md
python tools/validate_output.py 输出/八下复习讲义/{文件名}.md
```

标准 DOCX 导出命令：

```bash
python -c "import sys; sys.path.insert(0, 'tools'); from md2docx import process; process('输出/八下复习讲义/{文件名}.md', '输出/八下复习讲义/{文件名}.docx')"
python tools/compact_review_docx.py 输出/八下复习讲义/{文件名}.docx --output 输出/八下复习讲义/{文件名}.docx
```

注意：若复习讲义生成后需要导出 DOCX，必须先通过上述两个验证器，再执行 DOCX 导出。
