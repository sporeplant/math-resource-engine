# 复习课讲义整理技能

将多份新授课讲义合并整理为 1 份综合复习讲义。

---

## 一、输入与输出

**输入**：由用户在调用时指定（讲义编号或名称），不做任何默认假设。

**输出路径**：`输出/八下复习讲义/`（固定）。

**输出文件名**：由用户在调用时指定。

---

## 二、四模块结构

| 模块 | 内容 | 数量 | 编号方式 |
|------|------|------|----------|
| 1. 知识点与例题 | 源讲义的全部知识点，每个配 1 道「即学即练」 | 与知识点数一致 | 1．2．3．... |
| 2. 当堂练习 | 巩固练习 | **约 10 题** | 接续模块 1 |
| 3. 课后作业 | 综合提升 | **约 10 题** | 接续模块 2 |
| 4. 题目信息总览 | 表格（题目ID / 知识考察点 / 难度 / 入选理由） | 与前面题数一致 | — |

**横向规则**：全讲义一套顺序编号，不重置。

---

## 三、模块 1：知识点与例题

### 知识点选取

- 不合并，源讲义有几个知识点就保留几个。
- 完整保留每个知识点的概念讲解文字。

### 例题选取

- 来源：每个知识点的「即学即练」中选取 1 道。
- 数量：与知识点数量一致。

### 来源均衡原则

- 合并两份讲义时，题目选择应尽量在来源讲义之间保持均衡。
- 若后一份为专题型讲义，则按专题块、题型块或代表例题块尽量均衡分配，不强行按知识点等量拆分。
- 若某一来源题量明显不足，允许向另一来源倾斜，但不得让单一来源长期垄断例题、当堂练习或课后作业。

### 格式

```markdown
## 知识点{双位编号} 标题

知识点概念讲解文字（完整陈述，不留空）……

## 例题讲解

1．题目文字……

（1）第1小问

（2）第2小问

![](images/{哈希值}.扩展名)
```

---

## 四、模块 2：当堂练习

### 选題规则

优先级：① 典型例题；② 各知识点「即学即练」中未入选例题的部分；③ 综合练习中选取。

选题时遵循来源均衡原则，尽量覆盖两份讲义的主要题源。

### 数量

约 10 题，不足或超出时酌情调整。

### 编号

接续例题编号（如例题到第 3 题，当堂练习从第 4 题起）。

---

## 五、模块 3：课后作业

### 选題规则

优先级：① 未入选例题/当堂练习的变式题；② 有图的综合题；③ 原课后练习综合题。

选题时继续遵循来源均衡原则，尽量覆盖两份讲义的主要题源。

### 数量

约 10 题，不足或超出时酌情调整。

### 编号

接续当堂练习，全讲义一套顺序编号，不重置。

---

## 六、模块 4：题目信息总览表格

### 格式

```markdown
| 题目ID | 知识考察点 | 难度 | 入选理由 |
|--------|-----------|------|---------|
| 03讲-即学即练-1 | 坐标表示位置 | ★★★ | 覆盖坐标系建立的核心方法 |
| 04讲-第1题 | 坐标规律探索 | ★★★★ | 规律题代表，考查数形结合 |
```

### 题目ID 规则

格式：`源讲义名-模块-序号`，示例：

- `03讲-即学即练-1`、`03讲-即学即练-2`
- `03讲-题型01-典例1`
- `04讲-第1题`、`04讲-第2题`

### 难度等级

| 星级 | 含义 |
|------|------|
| ★ | 基础，巩固性题目 |
| ★★ | 基础偏中等 |
| ★★★ | 中等 |
| ★★★★ | 中等偏难 |
| ★★★★★ | 难，综合性强 |

### 入选理由

写明覆盖的知识点或选入目的（如"覆盖xx方法"、"规律题代表，考查数形结合"）。

## 七、格式细则

### 讲标题

```markdown
## 第 XX 讲 标题
```

- 使用 `##`（二级标题，不得使用一级标题 `#`）
- 格式固定：`## 第 XX 讲 内容标题`
- XX 为双位数字（01、02、14...）
- 标题后**不加任何元数据**（不加学习目标、思维导图等）

### 知识点标题

```markdown
## 知识点01 有序数对
```

- 使用 `##` 二级标题
- 格式：`## 知识点{双位编号} {知识点名称}`
- 双位编号顺序：01、02、03……
- 编号与标题之间有一个空格

### 概念讲解

- 普通段落文字，**不加粗、不加列表标记**
- 复习课中概念为**完整陈述**，不留空
- 行末使用中文句号 `。` 或全角句点 `．`

### 例题讲解标题

```markdown
## 例题讲解
```

- 固定写法 `## 例题讲解`，使用二级标题
- 每个知识点后紧跟一个"例题讲解"

### 题号格式

- 题号：`序号．`（**阿拉伯数字 + 全角句号**），如 `1．`、`2．`、`3．`
- 每个题号前有一个空行
- 小题标号：`（1）`、`（2）`、`（3）`
- 题目为普通段落文字
- 配图紧跟题后，图片前后各有一个空行

### 图片引用

```markdown
![](images/{哈希值}.扩展名)
```

- 格式：`![](images/{哈希值}.扩展名)`
- 路径：**相对路径** `images/`，与 .md 文件同目录
- 文件名：长哈希值（SHA256风格）
- 扩展名：`.jpg` 或 `.png`
- 图片前后各有一个空行

### 选择题选项

```markdown
A．$\frac{25}{4}$

B．$\frac{24}{4}$

C．2 或 $\frac{25}{4}$

D．2 或 $\frac{25}{4}$
```

- 选项字母 + 全角句号：`A．`、`B．`、`C．`、`D．`
- 每个选项**独立成行**
- 选项之间有一个空行
- 选项与题干之间有一个空行

### 数学公式

```markdown
行内：$a ^ { 2 } + b ^ { 2 } = c ^ { 2 }$
分数：$\frac { 2 5 } { 4 }$
角度：$9 0 ^ { \circ }$
三角形：$\triangle A B C$
下标：$_ { A B }$
```

- 统一使用 `$...$` 行内公式（不用 `$$` 块级公式）
- LaTeX 语法，字母和符号之间可以有空格
- 分数：`\frac {分子} {分母}`
- 角度：`\circ`
- 三角形：`\triangle`
- 下标：`_ { }`

### 空行规则

| 位置 | 空行数 |
|------|--------|
| 讲标题之后 | 1 个 |
| 知识点标题与概念讲解之间 | 1 个 |
| 概念讲解与例题讲解标题之间 | 1 个 |
| 例题讲解标题与第1题之间 | 1 个 |
| 例题之间 | 1 个 |
| 例题与配图之间 | 1 个 |
| 配图与下一模块之间 | 1 个 |
| 选择题选项之间 | 1 个 |
| 当堂练习/课后作业标题与第1题之间 | 1 个 |

## 八、配图搬运规则

### 搬运范围

只搬运被选用题目（模块 1/2/3 中实际引用的题目）引用的图片，不搬运整份源 `images/` 目录。

### 搬运流程

1. 识别所有被选用题目中的 `![](images/{哈希}.扩展名)` 引用。
2. 将对应的图片文件从源讲义 `images/` 目录复制到 `输出/八下复习讲义/images/`。
3. 输出讲义中的引用路径保持 `![](images/{哈希}.扩展名)` 不变，指向输出目录下的新图片。

### 示例

```
源：03讲.md → 引用 images/a1b2c3.jpg
操作：复制 a1b2c3.jpg → 输出/八下复习讲义/images/a1b2c3.jpg
讲义中引用：![](images/a1b2c3.jpg)（路径不变，自然指向输出目录下的新图片）
```

---

## 九、禁止事项

- 禁止修改源文件（知识库中的讲义）。
- 禁止虚构题目或改编题目。
- 禁止不按规则选取例题/当堂练习/课后作业。
- 禁止引用非同级 `images/` 的图片路径。
