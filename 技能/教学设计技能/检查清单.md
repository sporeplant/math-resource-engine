# 教学设计技能 检查清单

## Step01 知识分析

- [ ] 课型是否已正确判断
- [ ] 5 个分析维度是否全部覆盖
- [ ] 知识本质是否给出精确的数学定义
- [ ] 生长路径是否追溯到了认知转折点

## Step02 学习目标设计

- [ ] 三层结构是否完整（基础层/中间层/拓展层）
- [ ] 行为动词是否合法（无"理解/掌握/了解/体会/感受/知道"）
- [ ] 目标数量是否合理（每层 1-2 条，共 3-6 条）

## Step03 评价任务设计

- [ ] 每条目标是否对应至少一个评价任务
- [ ] 评价任务与目标的行为动词是否匹配
- [ ] 评价任务中的题目是否携带题源字段

## Step04 问题链设计

- [ ] 是否存在一个核心问题统领全课
- [ ] 子问题是否形成认知台阶
- [ ] 问题链是否服务学习目标
- [ ] 是否设计了认知冲突

## Step05 活动设计

- [ ] 活动是否服务核心问题（不是服务题目）
- [ ] 是否体现知识形成过程
- [ ] 是否避免直接给出结论
- [ ] 是否体现低起点、小台阶、快反馈
- [ ] 问题是否与活动步骤交织推进，形成台阶递进链
- [ ] 三层是否都有任务

## 活动质量检查

- [ ] 目标一致性：活动是否对应目标和评价
- [ ] 问题驱动：是否存在核心问题和问题链
- [ ] 认知发展：是否体现知识形成过程和认知冲突
- [ ] 分层设计：三层是否都有任务
- [ ] 数学深度：是否体现数学思想和方法提升
- [ ] 评价证据：是否有形成性评价

## Step06 作业设计

- [ ] 是否分层布置（基础层必做/中间层必做/拓展层选做）
- [ ] 作业量是否控制合理（基础层 ≤3 题、中间层 ≤2 题、拓展层 ≤1 题）
- [ ] 每道题是否携带题源字段

## Step07 质量检查

- [ ] 全部验证维度是否通过
- [ ] 目标-评价-活动链条是否完整
- [ ] 题源字段是否完整
- [ ] 状态是否为 `pending_human_review`

## 全局检查

- [ ] 所有题目都携带了 `source_id`、`source_type`、`question_id`
- [ ] 未生成课件或课堂提问参考答案
- [ ] 未写入具体学生姓名
- [ ] 课时总时长未超过 35 分钟
