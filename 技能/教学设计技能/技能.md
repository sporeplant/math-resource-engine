# 教学设计技能

## 1. 定位

本技能是教学设计的 **总控编排层（Orchestration Layer）**。

不替代任何已有技能的具体规则，也不直接设计目标、评价或活动。只负责：

- 编排 7 个设计步骤的执行顺序
- 明确每一步的输入来源、产出去向、负责技能
- 给定全局输出格式和跨步骤约束

---

## 2. 适用条件

| 条件 | 说明 |
|------|------|
| 触发命令 | `/lesson {课时名称}` |
| 输出路径 | `输出/{课时名}_教学设计.md` |
| 输出状态 | `review_status: pending_human_review` |
| 课时时长 | 不得超过 35 分钟 |

---

## 3. 调用顺序

```
Step01 知识分析
  ↓
Step02 学习目标设计
  ↓
Step03 评价任务设计
  ↓
Step04 问题链设计
  ↓
Step05 活动设计
  ↓
活动质量检查
  ↓
Step06 作业设计
  ↓
Step07 质量检查
  ↓
输出完整教学设计
```

### Step01 知识分析

| 项目 | 内容 |
|------|------|
| 负责技能 | `技能/知识分析技能/技能.md` |
| 输入 | 课时名称 |
| 执行 | 课型判断 + 5 维度知识分析（知识本质 / 生长路径 / 前置知识 / 后续知识 / 数学思想） |
| 产出 | 知识分析结果（不生成独立yaml文件） |
| 通过标准 | 5 个维度完整；知识本质有精确数学定义；生长路径追溯认知转折点 |

**课型判断规则**：从课时名称中提取关键词，匹配对应课型定义文件，确定标准流程框架。

| 课时名称特征 | 课型 | 读取文件 |
|-------------|------|----------|
| 含"复习课"且含"第X章"（非"单元"） | 复习课 | `知识库/课型/复习课.md` |
| 含"单元复习" | 单元复习课 | `知识库/课型/单元复习课.md` |
| 含"期中复习" | 期中复习课 | `知识库/课型/期中复习课.md` |
| 含"期末复习" | 期末复习课 | `知识库/课型/期末复习课.md` |
| 含"活动课"或"项目学习" | 数学活动课 | `知识库/课型/数学活动课.md` |
| 含"习题课"或"练习课" | 习题课 | `知识库/课型/习题课.md` |
| 含"试卷讲评"或"考试讲评" | 试卷讲评课 | `知识库/课型/试卷讲评课.md` |
| 其他（新概念/性质/定理第一课时） | 新授课 | `知识库/课型/新授课.md` |

### Step02 学习目标设计

| 项目 | 内容 |
|------|------|
| 负责技能 | `技能/学习目标技能/技能.md` |
| 输入 | Step01 知识分析结果 + `知识库/课程标准/` |
| 输出 | 分层学习目标（基础层 LO-B、中间层 LO-M、拓展层 LO-E，每层 1-2 条） |
| 通过标准 | 行为动词合法；无"理解/掌握/了解"等禁止词汇；三层完整 |

---

### Step03 评价任务设计

| 项目 | 内容 |
|------|------|
| 负责技能 | `技能/评价设计技能/技能.md` |
| 输入 | Step02 学习目标 + Step01 知识分析报告 |
| 输出 | 分层评价任务（AS-B/M/E，每条目标至少对应一个评价任务） |
| 通过标准 | 目标-评价一一对应；成功标准明确可观察；题源完整 |

---

### Step04 问题链设计

| 项目 | 内容 |
|------|------|
| 负责技能 | `技能/问题链设计技能/技能.md` |
| 输入 | 知识分析结果 + 学习目标 + 学情分析 |
| 执行 | 核心问题统领 → 子问题递进 → 认知冲突设计 |
| 产出 | 问题链（含核心问题、子问题链、认知冲突、最终结论） |
| 通过标准 | 一个核心问题统领全课；子问题形成认知台阶；问题链服务学习目标 |

---

### Step05 活动设计

| 项目 | 内容 |
|------|------|
| 负责技能 | `技能/活动设计技能/技能.md` |
| 输入 | Step02 学习目标 + Step03 评价任务 + Step04 问题链 + Step01 知识分析报告 |
| 参考规则 | `主控/规则.md` 规则9-10（学生主体、活动真实性）、`主控/教学法.md`（低起点/高目标/小台阶/快反馈）、`技能/分层提问技能/技能.md §8`（提问与内容呈现策略——低起点实现手段） |
| 输出 | 各环节活动设计（按课型标准流程组织，含活动类型、步骤、反馈、产出、题源） |
| 通过标准 | 活动以学生为中心；有反馈节点；无伪探究；支持分层 |

---

### 活动质量检查

| 项目 | 内容 |
|------|------|
| 负责技能 | `技能/教学活动质量检查技能/技能.md` |
| 输入 | Step05 活动设计 + 问题链设计结果 + 学习目标 |
| 执行 | 7 维度检查：目标一致性 / 问题驱动 / 认知发展 / 活动质量 / 分层设计 / 数学深度 / 评价证据 |
| 产出 | 质量检查报告（含通过项、问题项、修改建议、是否允许输出） |
| 通过标准 | 核心项全部通过；禁止未经检查直接进入作业设计 |

---

### Step06 作业设计

| 项目 | 内容 |
|------|------|
| 负责技能 | 当前由本技能直接编排 |
| 输入 | Step02 学习目标 + Step03 评价任务 + Step01 知识分析报告 + `知识库/练习册题库/` |
| 输出 | 分层作业（基础层必做 ≤ 3 题、中间层必做 ≤ 2 题、拓展层选做 ≤ 1 题） |
| 约束 | 每道题携带题源字段；优先从教材原文选题（练习栏目全部 + 习题A组1题 + 习题B组1题）；标注预估完成时间 |

---

### Step07 质量检查

| 项目 | 内容 |
|------|------|
| 负责模块 | 10 个验证器 + `主控/质量门.md` + `主控/审核协议.md` + 各技能检查清单 |
| 输入 | Step01~Step06 全部输出 + 活动质量检查报告 |
| 检查维度 | 学习目标 / 评价 / 活动 / 教学法 / 数学内容 / 学情适配 / 提问质量 / 一致性 / 教学设计整体 |
| 输出 | 质量检查报告 |
| 通过标准 | 全部维度通过，方可输出 `pending_human_review` |

---

### 设计顺序原则

依据 `主控/规则.md` 规则3（逆向设计原则）：

```
学习结果 → 评价证据 → 学习活动
```

即 Step02 → Step03 → Step05 是本流程的核心逻辑链，Step04 和 Step06 在活动设计前后穿插完成。

## 4. 输出格式

### 4.1 YAML 元数据

教学设计文件头部必须包含：

```yaml
---
content_type: lesson
lesson_id: ""
lesson_name: ""
command: lesson
workflow_version: v2
review_status: pending_human_review
source_files: []
created_at: ""
---
```

### 4.2 题源字段

教学设计中任何例题、练习、评价任务、作业题都必须携带：

```yaml
source_id: ""
source_type: textbook | exercise_bank | curriculum_repository
question_id: ""
```

禁止无来源题目进入教学设计。

### 4.3 禁止事项

- 禁止生成课件
- 禁止生成课堂提问参考答案
- 禁止写入具体学生姓名；教学设计中的提问只标注层级
- 禁止将未人工审核的教学设计标记为 `human_approved`

## 5. 流程原则约束

本技能所有步骤遵循 `主控/规则.md` 的以下全局规则：

| 规则 | 影响 |
|------|------|
| 规则1 新课标优先 | 所有步骤以课标为最高依据 |
| 规则2 教学评一致性 | Step02→03→05 链条不可断裂 |
| 规则3 逆向设计 | 禁止先设计活动再补目标 |
| 规则8 认知分层 | 所有步骤三层结构必须完整 |
| 规则13 低起点高目标小台阶快反馈 | 贯穿 Step01~Step06 及活动质量检查的设计决策 |
| 规则14 Skill职责隔离 | 每个步骤只调用对应技能，不越权 |
