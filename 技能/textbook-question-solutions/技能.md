# 教材问题解答技能

## 1. 定位

- 为指定教材课时建立完整、可追溯的参考解答文件。
- 覆盖教材中全部要求学生作出回答、判断、计算、证明、操作、作图、交流或书面表达的任务。
- 独立于“课堂提问参考答案技能”；本技能面向教材答案库，不依赖教学设计、课件、学生姓名或提问分层。
- 产物名称统一使用“参考解答”，不得声称为出版社标准答案。

## 2. 输入

| 输入项 | 来源 | 强制要求 |
|---|---|---|
| 教材课时信息 | `知识库/教材原文/教材原文_教材课时分配.md` | 唯一匹配目标课时 |
| 教材原文 | `知识库/教材原文/教材原文_{课时}.md` | 读取完整文件，不截取局部代替全文 |
| 教材图片 | 教材原文同级 `images/` | 所有“如图”题必须绑定原图 |
| 前置知识 | 本课及此前教材原文 | 主解法不得使用教材进度之外的概念或定理 |

## 3. 覆盖范围

按教材出现顺序提取并解答：

- 正文中的直接设问与任务指令；
- 观察与思考、大家谈谈、一起探究、做一做等栏目中的全部任务；
- 例题中的全部设问；
- 练习中的全部题目和小问；
- 习题 A 组、B 组、C 组及其他分组中的全部题目和小问；
- 数学活动、回顾与反思、项目式学习中的全部明确任务。

不能只按问号识别。以下表达同样属于任务：填空、判断、写出、求、证明、说明理由、猜想、画出、作图、观察、归纳、交流、调查、设计、操作、比较和评价。

## 4. 题目提取规则

1. 保持教材栏目、题目和小问的原始先后顺序。
2. 每个可独立作答的小问建立独立条目；共同题干只登记一次并建立父子关系。
3. 原题必须忠实引用，不得概括、改写或补充条件。
4. 教材原文引用中的全局禁止词不触发禁止词检查；AI生成的解答仍执行全局禁止词规则。
5. 图片位于题目前或题后时，按题号、图号、图注和邻接位置绑定。
6. 图片缺失、模糊或无法确认条件时暂停该题，不得猜图。

## 5. 稳定编号与题源

每题必须包含：

```yaml
question_id: "{课时编号}-{栏目代码}-{题号}-{小问号}"
source_id: "教材原文_{课时编号}_{课时名称}"
source_type: textbook
教材位置: "{栏目}第{题号}题第{小问号}问"
教材顺序: 1
任务类型: 计算题
认知层级: 基础层 | 中间层 | 拓展层
答案来源: 教材原文 | AI参考推导
```

栏目代码建议使用：`正文`、`观察思考`、`大家谈谈`、`一起探究`、`做一做`、`例题`、`练习`、`习题A`、`习题B`、`习题C`、`数学活动`、`回顾反思`。

认知层级只作逐题标注，不得据此重排教材问题。若某课时缺少某层题目，如实统计，不得补题。

## 6. 答案来源

- 教材已经完整给出解答：`答案来源: 教材原文`，忠实整理教材过程。
- 教材未给出完整解答：`答案来源: AI参考推导`，独立计算或论证并复核。
- 教材只给出部分过程：保留教材内容，将补充部分明确标注为 `AI参考推导补充`。
- 答案来源与正确性判断分开，不得把 AI 推导伪装成教材原文。

## 7. 分题型输出

### 7.1 计算与应用题

输出关键关系、计算步骤、结果和必要检验。应用题补充符合题意的结论表达。

### 7.2 证明题

输出已知、求证、证明过程和每一步依据。不得省略关键推理链。

### 7.3 作图题

输出作图步骤、作图依据、结果特征和检验方法。必要时引用教材原图，不以文字声称已经画图。

### 7.4 操作探究题

输出操作步骤、预期现象、可得结论和理由。不得只写结论。

### 7.5 观察、交流与开放题

输出参考作答要点、合理答案边界和一种示例表达。不得伪造唯一答案。

### 7.6 调查与实践题

输出实施方案、数据记录要求、处理方法和结果表达框架。涉及本班、本校或学生实际时，不得虚构真实数据。

## 8. 输出文件

路径：

```text
知识库/教材参考解答/教材参考解答_{课时编号}_{课时名称}.md
```

文件头：

```yaml
---
content_type: textbook_solution
lesson_id: ""
lesson_name: ""
command: 教材问题解答
workflow_version: v2
source_files: []
created_at: "YYYY-MM-DD HH:mm"
review_status: pending_human_review
---
```

本产物必须设置 `review_status: pending_human_review`，等待教师人工审核。教师审核通过后手动改为 `review_status: 审核通过`，教学设计工作流才能继续。

主体结构：

```markdown
# {课时名称} 教材问题参考解答

## 教材任务清单
| 教材顺序 | question_id | 教材位置 | 任务类型 | 图片依赖 | 答案来源 |

## 参考解答

### {教材位置}

题目元数据 YAML

**原题**：忠实引用教材原文

**参考解答**：按题型模板输出

## 覆盖统计
```

## 9. 数学与认知边界

- 数学概念、公式、定理、计算和推理零错误。
- 主解法优先采用教材本课已经呈现的方法。
- 不得使用后续课时才出现的概念、术语或定理替代教材表达。
- 多解题先给贴合教材进度的主解法；补充解法必须说明所需前置知识。
- 图形结论必须与教材图片和已知条件一致。

## 10. 禁止事项

- 禁止漏掉正文任务、栏目问题、练习、习题或小问。
- 禁止生成教材中不存在的新题、变式题或补充题。
- 禁止改变教材题目顺序或按认知层级重排。
- 禁止改写原题后当作教材原题。
- 禁止虚构班级、学校、调查或实验数据。
- 禁止为开放题制造唯一答案。
- 禁止在图片不可用时猜测条件。
- 禁止输出无题源字段的答案。
