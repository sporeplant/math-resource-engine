# 质量门

定义各阶段最低通过标准、自动拦截、强制回退与人工复核条件。

---

## 1. 各阶段最低通过标准

| 阶段 | 最低标准 |
|------|----------|
| 学习目标 | 所有目标使用合法行为动词；目标/学习结果表述无不可观察动词；三层结构完整 |
| 评价设计 | 每条目标对应至少一个评价任务；成功标准明确可观察 |
| 活动设计 | 活动以学生为中心；有反馈节点；无伪探究；支持分层 |
| 活动时间 | 核心任务预计耗时不超过40分钟；时间分项完整；双层时间一致；超时仅允许后移弹性或备用任务 |
| 课时设计 | 课时结构完整；有评价嵌入；有小台阶推进；有快反馈 |
| 教学法 | 不存在认知跳跃；无超过3分钟纯教师讲解；有快反馈机制 |
| 数学内容 | 核心概念定义准确；推理逻辑完整无断裂；公式符号规范 |
| 学情适配 | 不超纲；不超越教材和学生认知水平；基础层起点可及；拓展层有挑战；活动类型匹配班级水平 |
| 课件 | 上游教学设计已人工审核；A组结构合规全部通过；图片路径仅为 `./images/文件名`；B组违规 ≤ 2 项 |
| 一致性 | 目标-评价-活动链条无断裂；无无效活动；无无法评价的目标 |

---

## 2. 自动拦截条件

- 学习目标、学习结果、评价任务或成功标准中包含不可观察动词（理解、掌握、了解、熟悉、体会、感受、知道）
- outputs缺少分层结构（缺少基础/中间/拓展任意一层）
- 课时总时长超过 40 分钟
- 核心活动经班级画像或历史数据修正后预计超过40分钟
- 活动缺少 `time_priority`、`time_budget`，或时间分项与活动标题不一致
- 题目缺少 `source_id`、`source_type`、`question_id`
- 图片路径不是 `./images/文件名`
- `/courseware-collab` 读取的教学设计不是 `review_status: 审核通过`
- 评价任务与目标行为动词完全不匹配
- 活动描述中超过50%内容为教师讲授
- 数学内容出现明显错误
- 使用教材原文和课标要求之外、且学生当前认知水平难以承接的学科化术语或新概念
- 涉及国家、民族、历史、地图、社会事件、公共政策、人物机构或统计数据的内容存在明显事实错误、来源不明或价值导向风险

---

## 3. 强制回退条件

- 自动拦截条件触发 → 强制回退到对应设计Skill
- 审核结果为"不通过" → 强制回退
- 同一节点连续2次审核不通过 → 强制回退并记录
- 下游Skill检测到上游outputs缺失 → 强制回退到缺失节点

---

## 4. 人工复核条件

- 同一节点回退超过3次
- 审核结果为"有条件通过"但修改建议涉及重大调整
- 检测到knowledge中同一知识点存在无法自动解决的冲突
- 用户主动要求人工审核
- outputs包含课标中未涵盖的新增内容
- outputs使用教材未出现且未被课标要求支撑的超前概念术语
- outputs涉及国家、民族、历史、地图、社会事件、公共政策、人物机构或统计数据等敏感材料
- 教师或审核者认为内容可能存在价值导向、政策合规或公共表达风险

---

## 5. 数学错误零容忍规则

- 核心概念定义错误 → 直接拦截，强制回退
- 定理使用错误 → 直接拦截，强制回退
- 公式书写错误 → 拦截修正
- 推理过程逻辑断裂 → 标注后回退修正
- 上述错误二次出现 → 上报人工介入
