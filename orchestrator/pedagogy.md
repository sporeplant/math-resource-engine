# 初中数学教学法引擎（教学法.md）

# 文件定位

本文件定义：

# 系统级教学法机制（Pedagogy Engine）

用于指导：

- 学习结果设计
- 评价任务设计
- 学习活动设计
- 分层任务设计
- 提问设计
- 课堂反馈设计
- 作业设计

所有Skills必须遵循本文件教学法规则。

------

# 教学法核心框架

系统长期遵循：

# “低起点、高目标、小台阶、快反馈”

教学原则。

该原则不是口号。

而是：

# 可执行的教学认知机制。

系统必须将其转化为：

- 任务设计
- 活动设计
- 认知分层
- 课堂反馈
- 学习推进

中的具体行为。

------

# 第一部分：低起点机制（Low Entry）

# 核心目标

提升：

# 学生课堂参与率。

尤其：

- 学困生
- 基础薄弱学生
- 低自信学生

必须首先获得：

# “能够参与课堂”的最低认知条件。

------

# 系统认知原则

不同学生：

必须拥有不同认知起点。

禁止：

# “统一认知要求”

系统必须允许：

- 基础学生先进入观察层
- 中间学生进入理解层
- 优秀学生进入分析层

------

# 低起点任务设计规则

## 对基础学生：

优先设计：

- 能观察
- 能辨认
- 能复述
- 能模仿
- 能描述

等低阶认知任务。

------

## 禁止

禁止一开始直接要求：

- 综合分析
- 抽象推理
- 复杂建模

------

# 低起点活动特征

活动必须：

- 容易进入
- 可快速参与
- 能立即获得反馈
- 允许低风险表达

------

# 推荐活动形式

- 图形观察
- 数据读取
- 简单判断
- 口头复述
- 例题模仿
- 操作体验

------

# 系统执行要求

如果检测到：

# 学生基础较弱

系统必须自动：

- 降低认知入口
- 增加感性活动
- 增加模仿任务
- 增加支架提示

------

# 第二部分：高目标机制（High Expectation）

# 核心目标

提升：

# 高阶思维能力。

尤其针对：

- 学有余力学生
- 高认知学生
- 具备迁移能力学生

系统必须：

# 主动提高认知挑战水平。

------

# 系统认知原则

禁止：

# “只停留在记忆理解层”

系统必须推动学生进入：

- 分析
- 评价
- 创造

层级。

------

# 高目标任务设计规则

对于高水平学生：

必须增加：

- 多解法分析
- 方法比较
- 条件变式
- 开放问题
- 错误诊断
- 结构迁移

------

# 推荐高阶行为

- 分析解法合理性
- 比较不同策略
- 评价方法优劣
- 构造变式问题
- 推广结论
- 建立模型

------

# 系统执行要求

如果检测到：

# 学生具备较强认知能力

系统必须自动：

- 增加开放任务
- 增加变式训练
- 增加深度提问
- 增加迁移问题

------

# 第三部分：小台阶机制（Micro Scaffolding）

# 核心目标

建立：

# 连续、稳定、低跳跃的认知推进路径。

------

# 系统认知原则

学习推进必须遵循：

# 从具体到抽象

# 从模仿到理解

# 从单一到综合

认知路径。

------

# 禁止

禁止：

- 大跨度认知跳跃
- 一步到位抽象化
- 未建立表象直接推理

------

# 小台阶设计规则

系统必须：

将复杂认知任务拆分为：

- 微步骤
- 微任务
- 微推进

------

# 推荐推进路径

## 阶段1：感性认识

例如：

- 观察
- 操作
- 感知
- 比较

------

## 阶段2：表象建立

例如：

- 描述规律
- 归纳特征
- 建立联系

------

## 阶段3：抽象概括

例如：

- 形成概念
- 数学表达
- 符号化

------

## 阶段4：迁移应用

例如：

- 解决新问题
- 综合应用
- 结构迁移

------

# 系统执行要求

系统必须：

- 自动拆分认知跨度
- 自动增加脚手架
- 自动降低单步难度
- 保证前一步支撑后一步

------

# 第四部分：快反馈机制（Rapid Feedback）

# 核心目标

提升：

- 学习自信
- 学业自我效能感
- 持续参与意愿

------

# 系统认知原则

反馈必须：

# 高频、即时、过程化。

禁止：

# “只关注最终对错”

------

# 快反馈设计规则

系统必须同时提供：

## 一、学业反馈

针对认知目标：

即时反馈。

------

### 低阶目标达成

立即确认：

- 观察正确
- 描述准确
- 能参与表达

------

### 中阶目标达成

指出：

- 理解准确度
- 规律归纳情况
- 思维完整性

------

### 高阶目标尝试

给予：

- 思维点拨
- 推理引导
- 方法优化建议

------

## 二、学习状态反馈

系统必须关注：

# 学习投入变化。

------

# 必须识别

学生是否：

- 从旁观到参与
- 从沉默到表达
- 从被动到主动
- 从模仿到独立

------

# 必须及时强化

系统必须：

即时肯定：

- 微小进步
- 有效尝试
- 思维突破
- 表达勇气

------

# 系统执行要求

系统必须：

- 增加形成性评价节点
- 增加课堂即时反馈
- 增加过程性鼓励
- 缩短反馈周期

------

# 第五部分：认知分层机制

# 系统要求

所有教学资源必须：

# 默认存在认知分层。

禁止：

# “全班统一任务”

------

# 推荐分层结构

| 层级   | 目标           |
| ------ | -------------- |
| 基础层 | 能观察、能辨认 |
| 中间层 | 能解释、能归纳 |
| 拓展层 | 能分析、能创造 |

------

# 系统执行要求

系统必须：

自动生成：

- 分层问题
- 分层任务
- 分层评价
- 分层反馈

------

# 第六部分：教学活动设计机制

所有活动必须体现：

# “学生真实进行数学思考”

------

# 活动必须包含

- 观察
- 比较
- 归纳
- 推理
- 表达
- 验证

中的至少一种。

------

# 禁止活动

禁止：

- 形式化讨论
- 无目标合作学习
- 伪探究活动
- 纯展示型课堂

------

# 第七部分：系统级教学目标

本系统最终目标：

不是：

# “快速生成教案”

而是：

# “构建符合学生认知发展的数学学习系统”

所有资源生成，

必须服务于：

- 学生真实参与
- 学生思维发展
- 学生认知进阶
- 学生数学素养形成
