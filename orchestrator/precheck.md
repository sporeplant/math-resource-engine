# 前检查清单

任何 Skill outputs生成前，必须逐项确认以下内容。**所有检查项必须打勾后方可开始生成**。

---

## 通用检查项（所有任务类型必须执行）

### □ 协议已读取
- [ ] 已读取 orchestrator/skill-protocol.md，确认当前任务的强制步骤链
- [ ] 已明确当前任务对应的任务类型（教学目标类 / 评价设计类 / 活动设计类 / 教学设计类 / 课件设计类 / 教材分析类 / 习题分析类 / 提问质量审查类 / 临界生帮扶类）

### □ 教材参考解答已校验（`/lesson-collab`、`/courseware-collab`）
- [ ] 已定位 `knowledge/solutions/ch{章节号}/solution-{lesson_id}.md`
- [ ] 文件存在，且 `content_type: textbook_solution`
- [ ] `lesson_id` 与当前课时一致
- [ ] 已通过教材问题解答validators
- [ ] 缺失或任一校验失败时已终止本次任务，未自动补齐、降级推导或继续确认门

### □ skills定义已读取
- [ ] 已读取对应skills的skills.md 文件（skills/xxx/skills.md）
- [ ] 已明确该skills的分层规则
- [ ] 已明确该skills的行为动词规则（基础层 / 中间层 / 拓展层分别使用哪些动词）
- [ ] 已明确该skills的禁止事项

### □ outputs模板已确认
- [ ] 已读取skills定义中的outputs模板章节
- [ ] 已确认outputs结构的必填节
- [ ] 已确认outputs格式（Markdown / 其他）

### □ 自检规则已确认
- [ ] 已读取对应skills的检查清单.md
- [ ] 已明确自检通过的判定标准

## 不可跳过规则

**以下情况任意一条未满足，禁止开始生成，必须回退修正：**

1. □ skills调用协议.md 未读取 → 先读取，再开始
2. □ 对应skills的 skills.md 未读取 → 先读取，再开始
3. □ 对应skills的 检查清单.md 未读取 → 先读取，再开始
4. □ 课标文件未注入（需要课标的任务类型）→ 先注入，再开始
5. □ 学情文件未注入（需要学情的任务类型）→ 先注入，再开始
6. □ outputs模板未确认 → 先确认，再开始
