# 上下文优先级

定义注入知识的优先级顺序和超长时的裁剪规则。

---

## 优先级顺序

```
课标 ＞ 学情 ＞ 数学本质 ＞ 教学策略 ＞ 教材主线 ＞ 练习册题库
```

### 详细说明

| 优先级 | 内容 | 来源 |
|--------|------|------|
| 最高 | 课标要求 | 国家课程标准 |
| 高 | 学情数据 | students/ |
| 中高 | 数学本质分析 | math-essence/ |
| 中 | 教学策略建议 | teaching-strategies/ |
| 中低 | 教材内容结构 | 教材分析 |
| 任务关键 | 练习册题库、答案和逐题索引 | 评价、练习、例题、作业和讲评的正式题源 |

说明：练习册不改变概念形成和教材正文的主线顺序，但在“选题、评价证据、课堂练习、例题、作业、调度稿答案”中不再作为低优先级补充材料；对应课时存在练习册题库时，题库、答案和逐题索引必须完整注入并通过校验。

---

## 上下文超长裁剪规则

当注入内容超出上下文长度限制时，按以下顺序裁剪：

### 裁剪步骤

1. 首先裁剪练习册题目原文中的非选用题全文，但保留 `workbook-index`、被选用题和答案引用
2. 其次裁剪"教材"层级的非核心内容
3. 再裁剪"教学策略"中的非直接相关内容
4. 保留"课标"和"学情"的原始内容（核心不可裁剪）
5. 保留"数学本质"的核心定义（仅保留最相关的部分）

### 保留底线

- 课标中学业要求部分必须保留
- 学情中的分层数据（三层比例和特征）必须保留
- 被正式引用的练习册题目必须保留 `source_id/source_type/question_id`，且 `question_id` 必须可在 `knowledge/workbook-index/` 中查到
- 裁剪后必须保证有足够信息支撑当前Skill的正常执行

### 分Skill裁减策略

- learning-outcome：优先保留课标和学情
- assessment-design：优先保留课标和数学本质
- activity-design：优先保留学情和教学策略
